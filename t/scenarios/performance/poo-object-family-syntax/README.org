* POO Object Family Syntax Scenario

This scenario records the repair from repeated hand-written POO object
accessors and projections to a generated object-family declaration.

- =input.ss= keeps the old shape: one predicate, one accessor per slot, and one
  projection hand-written for every object family.
- =expected.ss= keeps the same object shape and public projection result, but
  declares the repeated family through =defpoo-object-family=.

The expected form is the project style for large stable POO object families:
slot ownership stays explicit, runtime payloads remain boundary projections, and
mechanical accessor/projection duplication is generated by Gerbil syntax.
