#+title: Runtime C Bindings

This directory owns the stable C ABI surface for runtime-language integration.

The files here are upstream bindings, not examples:

- =include/poo_flow_runtime_abi.h= defines the narrow runtime C ABI;
- =probe/poo_flow_runtime_abi_probe.c= is the current standalone ABI probe
  implementation.

The probe implementation is not the final runtime implementation. It fixes the
ABI and memory ownership contract before replacing the C body with a
Gerbil/Gambit compiled module.

The ABI includes opaque runtime graph plan handles. Runtimes can create a plan,
add nodes, bind nodes to runtime action symbols, add ordinary edges, add
conditional routes, bind state keys to reducer symbols, set a step limit, and
ask the control-plane binding for a graph handoff receipt without introducing a
second serialization protocol. The ABI also validates graph plans before
handoff so duplicate nodes, dangling endpoints, missing START/END boundaries,
unreachable nodes, and nodes that cannot reach END fail at the boundary.

Runtime language packages should consume this directory. They should not own or
fork the ABI definition.

Graph validation and handoff receipts include a stable plan digest so runtimes
can compare graph identity without receiving a second serialized graph format.

Run:

#+begin_src sh
uv run --project packages/python-runtime poo-flow-runtime-probe
#+end_src

Expected output:

#+begin_example
python-runtime-c-abi: ok
#+end_example
