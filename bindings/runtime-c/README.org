#+title: Runtime v0.1 C Bindings

This directory owns the installed pre-v1 C control-lane ABI.  It contains no
graph-plan compatibility API and exposes no Gerbil, Gambit, Python, or Scheme
heap type.

| Owner | Path |
|-------+------|
| Public C11 header | =include/poo_flow/runtime_v0.h= |
| POO-native ABI schema | =src/contract/runtime-v0-abi-schema.ss= |
| Control contract notes | =schema/runtime_v0_control.org= |
| Handle/lifecycle engine | =src/runtime_v0.c= |
| Pure C harness | =tests/runtime_v0_harness.c= |
| Hot/bulk harness | =tests/runtime_v0_batch_harness.c= |
| Installed consumer | =tests/runtime_v0_installed_consumer.c= |

The Bundle remains the semantic owner.  The C engine owns negotiation, opaque
handles, lifecycle state, checkpoint envelopes, owned bytes, and typed boundary
errors only.

Hot headers are fixed native structs with an explicit =header_stride=.  The v0
lane requires that stride to equal the generated header size, rejects
count/stride and payload-range overflow, and uses one =lower_bound= followed by
an ordered sequential merge for total batch lookup cost =O(log n + k)=.

Run the focused gates:

#+begin_src sh
bazel test //bindings/runtime-c:runtime_c_tests
bazel test //bindings/runtime-c:runtime_c_sanitizer_tests
bazel test //bindings/runtime-c:runtime_c_leak_test
bazel test //bindings/runtime-c:runtime_v0_batch_benchmark_smoke
bazel test //bindings/runtime-c:runtime_v0_gerbil_ffi_benchmark
bazel build \
  //bindings/runtime-c:runtime_c_static \
  //bindings/runtime-c:runtime_c_shared
#+end_src

=runtime_c_tests= includes a parity test between the Scheme-generated Bazel
contract outputs and the checked-in header/vector artifacts.  The contract rule
has an explicit dependency on =//:runtime_v0_schema=, so schema edits invalidate
generation and all downstream C consumers.

Bazel is the external runtime-C action graph.  =build.ss= remains the native
Scheme build authority and is not called or replaced by these C targets.

The installed contract is ABI v0.1 and Bundle schema
=poo-flow.organization-bundle.draft.3=.  Neither is an ABI v1 freeze claim.
