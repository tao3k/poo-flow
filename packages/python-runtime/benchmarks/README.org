#+startup: overview

* Scope
This benchmark compares native LangChain/LangGraph/CrewAI orchestration with semantically aligned POO Flow executor surfaces.

It does not measure real LLM latency, network tools, or durable storage IO. The deterministic actions model control-plane costs for policy routing, graph edges, dynamic handoff, reducers, checkpoint interrupt/resume, long graph scale, and stream/event consumption.

* Interpretation
- LangChain cases cover runnable composition, prompt/parser cost, branch routing, and tool/handoff selection.
- LangGraph cases cover the production graph controls we need to compare against: conditional edges, dynamic Send fanout, reducer aggregation, checkpoint-backed interrupt/resume.
- CrewAI cases cover sequential crew kickoff with agents, task context, and deterministic local LLM execution so the runtime-language boundary is tested without network or model latency.
- Long graph cases use a fixed 32-node linear graph to expose scale cost without introducing LLM or IO latency.
- Stream cases consume framework/runtime stream outputs so the benchmark is not limited to invoke-only happy paths.
- POO executor cases measure the current Python runtime graph semantics.
- The removed graph ABI is not reconstructed as a compatibility layer; runtime-v0 C ABI measurements remain in their dedicated benchmark record.

* Current Gaps Exposed
- The next benchmark expansion should add durable checkpoint persistence, larger graph-size sweeps, generated Scheme receipt -> Lean fact alignment, and a stable JSON artifact for CI trend comparison.
