#+title: LangChain/LangGraph C ABI Benchmark Report
#+startup: overview

* Scope
This benchmark compares native LangChain/LangGraph orchestration with the POO Flow runtime surfaces used to hand graph receipts to a runtime language.

It does not measure real LLM latency, network tools, or durable storage IO. The deterministic actions model the control-plane cost of AI Agent Flow shapes: policy routing, graph edges, dynamic handoff, reducers, checkpoint interrupt/resume, long graph scale, stream/event consumption, receipt materialization, and C ABI validation gates.

* Interpretation
- LangChain cases cover runnable composition, prompt/parser cost, branch routing, and tool/handoff selection.
- LangGraph cases cover the production graph controls we need to compare against: conditional edges, dynamic Send fanout, reducer aggregation, checkpoint-backed interrupt/resume.
- Long graph cases use a fixed 32-node linear graph to expose scale cost without introducing LLM or IO latency.
- Stream cases consume framework/runtime stream outputs so the benchmark is not limited to invoke-only happy paths.
- POO executor cases measure the current Python graph semantics without crossing the C ABI validation gate.
- POO C ABI receipt cases measure graph handle materialization and receipt creation with explicit handle close, so the benchmark does not hide handle lifetime leaks.
- POO C ABI validation failures are reported as gates and failures instead of being converted into fake performance numbers.

* Current Gaps Exposed
- The next benchmark expansion should add durable checkpoint persistence, larger graph-size sweeps, generated Scheme receipt -> Lean fact alignment, and a stable JSON artifact for CI trend comparison.
