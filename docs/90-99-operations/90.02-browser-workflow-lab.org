This document is an executable presentation surface for the Poo Flow browser
runtime.  The Scheme source remains the canonical workflow definition; the web
view projects runtime events into an interactive graph without embedding a
Scheme compiler in the WASM runtime.

** Contribution policy

The workflow chooses a profile, strategy, policy, and scenario before producing
a reusable, composable, documented, and proof-addressable contribution.

#+begin_src typst
$
op("Contribution")(c)
=>
op("Profile")(c)
or op("Strategy")(c)
or op("Policy")(c)
or op("Scenario")(c)
$
#+end_src

#+begin_src typst
$
op("Reusable")(c)
=>
op("Named")(c)
and op("Composable")(c)
and op("Documented")(c)
and op("ProofAddressable")(c)
$
#+end_src

** Executable workflow

The browser adapter consumes the runtime event projection of this workflow.
The source is kept visible so the graph never becomes an opaque replacement for
the actual Scheme program.

#+name: browser-contribution
#+attr_html: :data-poo-flow true
#+begin_src scheme
(workflow browser-contribution
  (step receive-request
    (emit 'request-received))
  (step select-profile
    (requires receive-request)
    (emit 'profile-selected))
  (step compose-policy
    (requires select-profile)
    (emit 'policy-composed))
  (step execute-scenario
    (requires compose-policy)
    (emit 'scenario-executed))
  (step verify-proof
    (requires execute-scenario)
    (emit 'proof-verified))
  (step publish-contribution
    (requires verify-proof)
    (emit 'contribution-published)))
#+end_src

** Acceptance checklist

- The workflow block retains readable Scheme source.
- Running the projection opens an interactive, fitted graph.
- Runtime errors remain local to the workflow figure.
- LaTeX follows the active Documents palette; the neighboring rendering lab
  remains the dedicated Typst qualification surface.
- Leaving the page releases observers, workers, and React roots.
