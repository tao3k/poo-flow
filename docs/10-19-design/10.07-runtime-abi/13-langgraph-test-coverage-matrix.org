* Benchmark Gate

The executable benchmark control surface is:

#+begin_src sh
uv run --project packages/python-runtime python -m poo_flow_runtime.benchmarks langgraph-alignment \
  --iterations 250 --fanout 24
#+end_src

It emits one line receipt per scenario:

- =sequence-composition=
- =dynamic-send-fanout=
- =subgraph-composition=
- =thread-state-facade=

These scenarios are deliberately stronger than simple API parity. They verify
that POO Flow can compose graph construction, dynamic fan-out, subgraphs, and
durable state access without importing LangGraph or moving policy semantics out
of Scheme.

* Next Adaptation Rule

When adapting a LangGraph test:

1. If it checks local runtime semantics, port it into the Python runtime tests.
2. If it checks backend-specific checkpoint storage, map it to the durable
   adapter contract instead of copying SQLite/Postgres behavior.
3. If it checks SDK, server, Cloud, CLI deploy, LangSmith, or model-provider
   behavior, document it as outside the POO Flow runtime ABI boundary.
4. If it reveals a stronger composition pattern, add it to the benchmark
   receipt surface rather than copying framework-specific setup.
