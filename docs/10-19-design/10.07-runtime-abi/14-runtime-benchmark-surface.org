#+title: Runtime Benchmark Surface

* Status

The graph-domain benchmarks remain historical/domain evidence.  Old C ABI hot
path claims are retired and do not qualify runtime v0.1 or the future ABI v1.

* Intent

Benchmarks are package module surfaces, not pyproject script expansion.

The benchmark layer must prove two things:

- LangGraph-compatible runtime semantics are covered by executable tests.
- POO Flow can go beyond parity through stronger functional composition, durable policy integration, nested subgraph projection, and measured C ABI hot paths.

* Composition suite

The composition suite currently emits line-oriented receipts for:

- strategy-combinator-pipeline
- durable-fanout-strategy
- nested-subgraph-stream-projection
- cabi-status-hot-call
- validation-workflow-round-trip

The C ABI scenarios intentionally separate the loaded binding hot path from the full validation workflow.  The hot path measures the low-level ABI call after the dylib is loaded.  The workflow path measures manifest validation and graph handoff as a higher-level runtime operation.

* AnyIO micro qualification

The current candidate validates and prepares one executor at each public
=abatch= boundary, reuses that prepared executor only within the batch, and
refreshes preparation for the next batch.  There is no permanent program
cache.  The immediately preceding same-semantics failing receipt is retained
below as the before baseline.

The canonical micro receipt uses schema
=poo-flow.anyio-runtime-benchmark.v1= and a native AnyIO bounded-concurrent
reference with the same capacity and action as the
=RuntimeGraphProgram.abatch= candidate.  The earlier 83.08-second serial-sleeper
receipt is rejected because it compared unequal execution semantics and spent
most of its wall time in synthetic sleep.

* Burst lifecycle qualification

** Current prepared-batch receipt

The production-shaped burst suite keeps the case mix explicit: case IDs rotate
through serial-only, parallel-only, and hybrid shapes.  Parallel fanout remains
action-internal; this is not a graph-native parallel-node claim.  Public
=abatch= validates and prepares one executor, bounded workers reuse it within
that batch, and the next public batch refreshes preparation without a permanent
cache.  Live worker tasks remain =O(capacity)=, while inputs and ordered results
remain =O(N)=.

** Before baselines

The production-shaped owner uses schema
=poo-flow.burst-lifecycle-benchmark.v1=.  Its case mix supports serial-only,
parallel-only, and hybrid cases.  Parallel work is action-internal fanout/join;
this receipt does not claim graph-native parallel fanout.  The measured arrival
profile is instant submission.

With an explicit capacity, the shared ordered worker pool keeps live tasks at
=O(capacity)= while input and ordered-result storage remain =O(N)=.  The worker
pool preserves input order and retains legitimate =None= results through a
private sentinel.
