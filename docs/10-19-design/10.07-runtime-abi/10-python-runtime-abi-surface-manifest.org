#+title: Python Runtime ABI Surface Manifest

* Purpose

=poo-flow.python-runtime.abi-surface.v1= is a machine-readable line manifest for
runtime-language bindings.

It does not replace Scheme-owned policy manifests. Scheme remains the owner of
POO-native durable policy and runtime envelope semantics. The Python runtime
package publishes the ABI surface it consumes and exposes so another runtime
binding can confirm compatibility before binding concrete backend details.

* Carrier

The manifest is exported from the Python package:

- =runtime_abi_surface_manifest=
- =runtime_abi_surface_manifest_lines=
- =runtime_abi_surface_manifest_string=
- =runtime_abi_surface_manifest_bytes=
- =coerce_runtime_abi_surface_manifest=
- =poo-flow-runtime-abi-surface=

The byte carrier follows the same line-oriented style as other runtime
receipts. It deliberately avoids JSON because this surface is a small runtime
contract, not a nested policy document.

The command-line entrypoint is intended for non-Python runtime bindings that
want to inspect the surface before loading Python modules or generating FFI
glue.

It can also validate a captured manifest:

#+begin_src sh
poo-flow-runtime-abi-surface --validate abi-surface.manifest
poo-flow-runtime-abi-surface --validate -
#+end_src

Validation emits =poo-flow.python-runtime.abi-surface.validation.v1=. A valid
payload returns status =ok= and exit code 0. An invalid payload returns status
=fail= and exit code 1.

* Fields

- =schema=
- =owner=
- =policy-owner=
- =runtime-role=
- =entrypoints=
- =durable-adapter-facade=
- =manifest-schemas=
- =receipt-schemas=
- =gates=

Runtime bindings should validate these fields before assuming the package
supports a given adapter facade or durable manifest schema.

* Validation

=coerce_runtime_abi_surface_manifest= parses and validates the line manifest.
It rejects:

- unsupported schema ids
- unsupported owner or policy owner
- unsupported runtime role
- missing stable entrypoints
- missing durable adapter facade methods
- missing Scheme manifest schemas
- missing receipt schemas
- missing developer gates

Other runtime languages do not need to call this Python function directly, but
their native validator should enforce the same checks before generating FFI or
backend glue.

* Contract gate

The Python runtime package includes focused tests for this public surface:

- =test_runtime_abi_surface.py=
- =test_runtime_public_surface.py=

These tests fail when package exports drift, when durable adapter methods are
removed, or when the machine-readable manifest stops advertising the stable
entrypoints needed by Rust, Marlin, Python, or another runtime language.

* Validation Entrypoints

The manifest includes a =validation-entrypoints= field. It lists the public
Python package functions that validate ABI manifests and validation receipts:

- =coerce_runtime_abi_surface_manifest=
- =coerce_runtime_abi_surface_validation_receipt=
- =runtime_abi_surface_self_test=
- =runtime_abi_surface_self_test_receipt=
- =runtime_abi_surface_validation_receipt=

Downstream runtime harnesses can discover these names from the same line
manifest that lists runtime entrypoints and receipt schemas. This keeps the
adapter contract machine-checkable without turning Python into the policy
strategy layer.
