#+title: AC-07 Python Production CFFI Runtime Qualification Receipt
#+date: 2026-07-12

* Decision

AC-07 is qualified as the production Python adapter for runtime v0.  The wheel
contains an out-of-line CFFI API extension and a qualified platform library.
Production graph execution requires a negotiated native context; the pure
Python backend is available only through explicit reference constructors.

=abi-v1-frozen? #f=

* Boundary receipt

- Scheme remains the ABI schema and typed projection owner.
- Python receives, but never synthesizes, Bundle digest, epoch, schema, and
  canonical packet.
- Graph =plan_digest= remains graph-domain evidence and is never reinterpreted
  as Bundle identity.
- No =ctypes= path or public graph C ABI exists.
- The private producer header was removed.  Public =publish_batch= validates an
  entire batch before mutation and appends in =O(k)=.
- The unreferenced 1500-line legacy CABI benchmark was deleted rather than
  preserved or split; dependency and manifest search found zero external users.

* Native lifecycle receipt

- Dynamic library loading: bundled path and explicit
  =POO_FLOW_RUNTIME_V0_LIBRARY= override pass.
- ABI v0.1/schema/capability negotiation: pass.
- Instance, profile, Bundle, session, arena, and lease ownership: pass.
- Partial construction cleanup, context manager, and idempotent close: pass.
- Writable caller buffer strong reference and 16-byte alignment: pass.
- Typed two-event =publish -> poll -> submit -> ack= roundtrip: 2 published,
  2 produced, 2 accepted, 0 rejected, bitmap =0x03=.
- Ack followed by arena generation recycle: pass.

* Production policy

- =RuntimeGraphRuntime()= selects native production policy.
- Missing or closed negotiated context fails before graph validation/execution.
- =RuntimeGraphRuntime.reference()=,
  =RuntimeGraphProgram.reference()=, and
  =RuntimeGraphBuilder.compile_reference_program()= are explicit test/reference
  routes, not automatic fallback.
- Existing graph receipt, checkpoint, store, tool, stream, and subgraph
  semantics remain Python-local strategies.
