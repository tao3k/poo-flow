#+title: AC-08 Cedar, Token, and Evidence Roots Implementation Plan
#+date: 2026-07-12

* Delivery

Implement RFC 123 in bounded vertical slices and commit only after complete
AC-08 qualification.  ABI v1 remains unfrozen.

* Step 1 — Scheme POO token and roots

- Add deterministic semantic-root and one-shot AuthorizedEffectToken objects.
- Add pure issue, validate, reserve, consume, reject, revoke, and indeterminate
  transitions with immutable receipts.
- Bind Bundle/policy/decision/intent/runtime/session/lease/arena/time/revocation.

* Step 2 — Tool-control and proof projection

- Require a valid token at the tool-calling control boundary.
- Project decision and consumption receipts into proof-gate facts.
- Add mismatch, replay, reuse, substitution, stale epoch, and TOCTOU fixtures.

* Step 3 — Runtime-v0 strict mediation

- Add atomic nonce reservation, lease binding, token consumption, and execution
  root commit to the kernel.
- Invoke downstream native adapters through a synchronous C vtable only after
  token/root/lease validation and nonce reservation.  The invocation exposes
  strided kernel headers and the caller-owned payload arena as zero-copy views.
- Treat adapter failure or unknown observation as a successful mediation receipt
  with =Indeterminate= outcome, spent nonce, and unchanged execution root.
- Make Strict default; negotiate Batched explicitly; forbid Diagnostic execute.

* Step 4 — Python CFFI and differential vectors

- Project the mediation API through installed C artifacts and CFFI.
- Run shared Scheme/C/Python vectors and crash/restart root reconciliation.

* Step 5 — Qualification

- Run AC regressions, security/durability/fork tests, Strict/Batched performance,
  owner matrix, and L2/L3 evidence gates.
- Write the receipt with =abi-v1-frozen? #f= and commit one complete feature.
