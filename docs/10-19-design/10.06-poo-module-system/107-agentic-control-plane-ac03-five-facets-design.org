#+title: AC-03 Five Typed Facets and Cross-Facet Validation Design

* Status

Approved in conversation on 2026-07-11.  Implementation and focused
qualification are complete.

* Objective

AC-03 upgrades the Canonical Organization Bundle to contain all five mandatory
typed semantic facets defined by RFC 91: Organization, Authority, Context,
Protocol, and Evidence.  They share one identity universe, schema, epoch,
lineage, digest, and validation result.

The migration is a direct canonical cutover.  There is no dual Bundle shape,
compatibility parser, missing-facet fallback, or last-writer-wins merge.

* Typed facet contract

Every facet is a POO-native object containing:

| Slot | Meaning |
|------+---------|
| =kind= | facet-specific stable kind |
| =schema= | facet schema version |
| =facet-name= | one of the five canonical facet names |
| =entities= | typed entities owned or projected by the facet |
| =relations= | typed semantic relations |
| =constraints= | typed invariant declarations |

All five facets are required.  A facet with no members is represented by its
named typed-empty constructor.  =#f=, a missing slot, or a raw empty alist is
invalid.

* Facet boundaries

** Organization

Owns principals, roles, agents, resources, artifacts, and organization parent
relations.  Organization hierarchy does not grant authority.

** Authority

Owns capabilities, grants, delegations, attenuation, revocation, and leases.
Every authority subject and capability reference resolves through the shared
identity universe.

** Context

Owns visibility, information-flow constraints, provenance, and declassification
authority.  Context visibility cannot exceed effective delegated authority.

** Protocol

Owns lifecycle states, messages, transitions, effects, cancellation, retry,
recovery, and checkpoint obligations.  Effects reference Authority capabilities.

** Evidence

Owns obligations, receipts, witnesses, causal links, verifiers, and proof
artifact references.  AC-03 models these as semantic requirements; it does not
generate Lean proofs or runtime evidence.

* Bundle cutover

The Bundle constructor accepts a manifest/epoch plus exactly five typed facets.
Canonical normalization emits a versioned facet sequence in the fixed order:

#+begin_src scheme
(organization authority context protocol evidence)
#+end_src

The prior flat section constructor and canonical schema are removed.  AC-01 and
AC-02 fixtures migrate in the same feature.  Empty Protocol and Evidence facets
remain explicit typed objects in the minimal parent/child/tool vertical slice.

* Shared identity universe

The identity universe is derived deterministically from the entities declared
across all facets.  Duplicate typed identities, incompatible kinds for one id,
and unresolved references fail closed.  Facets do not own independent epochs,
digests, or version lineages.

* Cross-facet validation

The validator runs facet-local checks first and cross-facet checks second.  It
returns one ordered validation receipt.

Required cross-facet rules:

1. Authority subjects resolve to Organization entities.
2. Authority capability references resolve uniquely.
3. Context subjects resolve to Organization entities.
4. Context visibility cannot exceed effective delegation.
5. Protocol effects reference capabilities authorized by Authority.
6. Protocol participants and message endpoints resolve to Organization entities.
7. Evidence obligations reference a valid subject, protocol item, effect, or
   Bundle identity.
8. Every facet has the required schema and canonical facet name.
9. All identities are unique in the shared typed identity universe.
10. Missing or raw-empty facets fail before cross-facet rules run.

Cross-facet diagnostics carry facet, semantic path, code, expected constraint,
and observed category.  Their ordering is stable by facet order, then semantic
path, then code.

* Kernel integration

AC-02 Kernel functions remain unchanged at their public boundary.  They consume
the upgraded Bundle through AC-03 normalization and validation.  A Bundle with
a facet-local or cross-facet failure cannot become =validated= or =advanced=.
Identity conflicts and stale epochs still fail before next-Bundle validation.

* Qualification

The feature is complete only when focused tests prove:

1. all five non-empty/typed-empty facets normalize deterministically;
2. missing, =#f=, and raw-empty facets fail closed;
3. construction order does not affect Bundle identity;
4. duplicate and incompatible shared identities fail;
5. authority subject/capability references are checked;
6. context visibility is bounded by delegation;
7. protocol participants and effects are checked;
8. evidence references are checked;
9. diagnostics have stable facet/path/code order;
10. the minimal parent/child/tool Bundle remains valid with explicit empty
    Protocol and Evidence facets;
11. AC-02 open, validate, no-op, conflict, stale, rejection, and advance gates
    remain green after fixture migration;
12. Org parsing and =git diff --check= pass.

* Out of scope

AC-03 does not implement C ABI layouts, Python runtime behavior, Cedar policy
evaluation, Lean proof generation, persistence, mutable sessions, or runtime
effect execution.

* Delivery

Design, plan, semantic implementation, migrated AC-01/02 fixtures, focused AC-03
tests, owner matrix, and qualification receipt form one complete feature commit.
Pre-existing build and CLI-support changes remain outside it.
