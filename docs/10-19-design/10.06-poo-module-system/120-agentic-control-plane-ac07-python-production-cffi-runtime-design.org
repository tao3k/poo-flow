#+title: AC-07 Python Production CFFI Runtime Design
#+date: 2026-07-12

* Status

Approved in conversation on 2026-07-12.  Implemented and qualified by receipt
122.  Runtime v0 gained the approved public atomic producer amendment; ABI v1
remains deliberately unfrozen.

* Objective

AC-07 proves that the public runtime v0 C ABI supports a production-capable
Python runtime without making Python a semantic owner.  =RuntimeGraphProgram=
remains the graph-domain execution owner; =RuntimeGraphRuntime= remains its
public runtime context boundary.  One internal, out-of-line CFFI API-mode
adapter becomes the mandatory production backend.

The pure Python implementation remains an explicitly selected reference and
test backend.  Native loading or negotiation failure is fatal in production;
there is no silent fallback.

* Non-goals

- Do not restore the retired public graph C ABI.
- Do not add a JSON contract owner.
- Do not retain a =ctypes= production or diagnostic lane.
- Do not move graph plans, checkpoints, stores, context, messages/tools,
  subgraphs, builder loops, or prebuilt strategies into POO Flow semantic
  ownership.
- Do not freeze ABI v1.
- Do not add Rust, TypeScript, or other downstream runtime adapters.

* Chosen approach

Use an out-of-line CFFI API extension generated from the installed public
header.  The compiler, rather than Python runtime guesses, validates layout,
types, and calling convention.  The extension links a platform-native library
bundled in the wheel.  An explicit environment variable may override that
library for development or controlled deployment, but the replacement must
pass the same negotiation and identity gates.

CFFI ABI mode and a handwritten CPython extension are rejected.  ABI mode
defers layout drift to runtime and weakens proof.  A handwritten extension adds
reference-counting and buffer-lifetime machinery without improving the public
contract.

* Ownership boundaries

- Scheme POO objects remain the ABI schema and typed projection owners.
- Generated headers and CFFI declarations are projections, never independent
  contract sources.
- The C runtime owns handles, arenas, leases, generations, ordered pending
  windows, native lifecycle, and batch receipts.
- Python owns graph-domain strategies and converts their typed results to the
  public native submission contract.
- Python consumes only published projection artifacts.  A production wheel
  requires no Gerbil installation or private Gerbil bypass.
- Python receives a formal Bundle descriptor from its caller.  It cannot derive
  Bundle identity from a graph-plan digest or synthesize canonical Bundle bytes.

* Required runtime v0 amendment

AC-06 exposed consumer =poll_batch= and =submit_batch= operations while leaving
event publication behind a private header.  That boundary cannot support an
independent downstream runtime using installed artifacts only.  Before the
Python adapter is connected, runtime v0 adds a public =publish_batch= producer
operation with these invariants:

- installed public header only; the private producer header is removed;
- one FFI crossing for =k= fixed-stride headers;
- full descriptor, stride, count arithmetic, sequence, generation, and payload
  validation before the first queue mutation;
- atomic failure: malformed or out-of-capacity batches publish zero items;
- monotonic append cost =O(k)= and no per-item allocation;
- a result reports published count and last sequence.

This amendment changes only ABI v0, which remains unfrozen.  It does not assign
event-kind semantics to Python or expose graph plans through the C ABI.

There are no native callbacks into Python.  Poll and submit remain explicit
batch boundaries, avoiding callback/GIL crossings and per-event FFI calls.

* Resource and buffer lifetime

Each native handle has one Python owner object.  Owners implement explicit,
idempotent =close()= and context-manager protocols.  Finalization is only a
last-resort safety net and cannot define correctness.

An arena is created from a writable Python buffer through =ffi.from_buffer()=.
The arena owner holds a strong reference to that buffer for the complete native
registration lifetime.  While any lease is outstanding, the arena cannot be
resized, recycled, or released.  Submission is followed by explicit
acknowledgement before generation advancement.

No Python, Scheme, Gerbil, or Gambit heap pointer crosses the ABI.  Payloads are
addressed by checked offset and length inside the registered arena.

* Error model

Native status codes map to a stable Python exception hierarchy.  Exceptions
carry operation, status, ABI version, Bundle digest, and session identity, but
never copy payload content.

- ABI, schema, capability, architecture, or bundled-library mismatch fails at
  backend startup.
- Stale handles, stale generations, busy arenas, and malformed descriptors map
  to precise exception types.
- Partial acceptance returns a typed per-item result and preserves accepted
  items; it is not collapsed into one exception.
- Missing or invalid native libraries fail closed.  Production mode never
  silently selects the reference backend.
- Cleanup runs on normal return, Python exception, cancellation, and partial
  construction failure.

* Wheel and native-library delivery

Platform wheels bundle the qualified runtime v0 dynamic library and the
out-of-line CFFI extension.  Loader paths are package-relative:

- macOS: =@loader_path=;
- Linux: =$ORIGIN=;
- Windows: package-local DLL directory.

=POO_FLOW_RUNTIME_V0_LIBRARY= is the single explicit override.  Override does
not bypass ABI/schema/capability negotiation.  Universal ABI-guessing wheels
are forbidden; each platform and architecture artifact is qualified
independently.

The CFFI declaration is generated from the formal installed header during the
build pipeline and checked for freshness in the repository.  Wheels install
and run without Gerbil.  An sdist build requires an explicit qualified header
and library input and fails with a diagnostic when they are absent.

* Compatibility policy

=RuntimeGraphRuntime=, graph materialization, trace, builder, and public
graph-domain values remain the Python-facing surface.  The native adapter is an
internal execution backend, not a revived public graph ABI.

There is one production path: out-of-line CFFI API mode.  No =ctypes= alias,
legacy import, automatic backend fallback, or compatibility wrapper is
provided.  The reference backend is available only through an explicit test or
reference configuration.

* Qualification matrix

** Contract

- Generated CFFI declaration freshness.
- C/Python structure size, offset, alignment, enum, and vector agreement.
- ABI/schema/capability negotiation.
- Public-surface proof that =ctypes= and retired graph ABI exports are absent.

** Lifecycle and fault

- Context-manager and explicit close on success and exception paths.
- Idempotent close, stale generation, outstanding lease, early recycle, and
  partial construction.
- Buffer strong-reference and memory-pressure qualification.
- Crash/restart and checkpoint restore.
- Long-run concurrent sessions with injected native and Python exceptions.

** Differential

Run the same materialized graph through the explicit reference backend and the
native backend.  After declared nondeterminism normalization, trace, receipt,
Bundle digest, disposition, and checkpoint identity must agree.

** Wheel

- Install and execute in an isolated environment without Gerbil.
- Qualify both bundled-library and explicit-override loading.
- Reject missing, wrong-architecture, wrong-ABI, wrong-schema, and
  wrong-capability libraries.
- Prove the loaded library is the artifact named by the wheel receipt.

** Performance

- Batch sizes: 1, 8, 32, 128, and 1024.
- Python/C crossings remain fixed per batch, not per event.
- Arena payload access remains zero-copy.
- The C steady-state path remains zero-allocation per item.
- Ordered-window lookup remains =O(log n + k)=.
- Performance gates execute against the library loaded from the built wheel,
  not an uninstalled repository artifact.

* Exit criteria

1. Production =RuntimeGraphRuntime= uses out-of-line CFFI by default and fails
   closed when native startup cannot be proved.
2. Pure Python execution requires explicit reference/test selection.
3. No =ctypes= or retired graph ABI route remains.
4. Wheels contain and load the qualified native library without Gerbil.
5. Native and reference traces agree after declared normalization.
6. Lifecycle, fault, long-run, crash/restart, wheel, and performance gates pass.
7. Receipts bind the wheel, native library, header projection, Bundle digest,
   trace, and runtime disposition.
8. Qualification reports =abi-v1-frozen? #f=.

* Delivery boundary

The subsequent AC-07 implementation plan may change only the Python runtime
package, its focused tests and wheel build, runtime-C packaging hooks needed to
publish existing projection artifacts, and AC-07 status/qualification docs.
Existing unrelated CLI/build/CI changes remain outside the feature commit.
