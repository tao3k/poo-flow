#+title: AC-05 Runtime v0.1 C ABI Control Lane Design

* Status

Approved in conversation on 2026-07-12.  This document is the written design
gate.  Implementation and qualification remain pending.

* Objective

AC-05 introduces a formally installed pre-v1 C runtime interface for negotiated
control-plane lifecycle operations.  The public surface is versioned as v0.1:
it is production-quality and fully tested, but incompatible changes remain
permitted before the complete ABI v1 freeze.

The prior graph ABI is removed directly.  There is no compatibility header,
symbol alias, deprecated wrapper, dual route, or Python binding exception.

* Version policy

The public pre-v1 surface uses:

- symbol prefix =poo_flow_runtime_v0_=;
- ABI major/minor =0.1=;
- installed header =poo_flow/runtime_v0.h=;
- installed library =libpoo_flow_runtime_v0=.

Runtime v0.1 negotiates the current
=poo-flow.organization-bundle.draft.3= schema explicitly.  It does not imply
Bundle v1 or ABI v1 stability.  The final v1 names are reserved until Bundle,
Kernel, shadow qualification, control/hot/bulk lanes, Python CFFI, Cedar,
Lean/Evidence, fault, performance, and complete conformance gates pass.

* Implementation ownership

AC-05 uses a schema-derived contract plus a C handle/lifecycle engine.

- The validated Gerbil/POO Bundle remains the semantic owner.
- One control schema derives the public header, Gerbil control-packet
  projection, and canonical conformance vectors.
- The C engine owns only ABI resources, descriptor validation, negotiation,
  lifecycle mechanics, handle generations, buffers, and typed failures.
- The C engine does not reimplement Organization, Authority, Context, Protocol,
  Evidence, Cedar, or proof semantics.
- A pure C consumer links only installed public artifacts and never calls a
  private Gerbil/Gambit symbol.

* Public lifecycle API

The control lane contains:

- =poo_flow_runtime_v0_instance_create/release=;
- =poo_flow_runtime_v0_negotiate=;
- =poo_flow_runtime_v0_bundle_open/release=;
- =poo_flow_runtime_v0_session_open/cancel/close/release=;
- =poo_flow_runtime_v0_session_checkpoint=;
- =poo_flow_runtime_v0_session_restore=;
- =poo_flow_runtime_v0_owned_bytes_release=;
- =poo_flow_runtime_v0_error_message=.

Close terminates protocol state; release reclaims a handle.  These operations
are never aliases.  Cancel is an idempotent request.  Close requires an explicit
outstanding-work disposition.  A session binds to one Bundle identity and
epoch and cannot switch silently.

Checkpoint produces ABI-owned bytes containing Bundle identity, Bundle epoch,
session sequence, lifecycle state, and integrity data.  Restore validates the
packet and creates a new session handle; it never revives an old handle.  AC-05
does not own file, database, or cross-process persistence.  A runtime adapter
stores and returns checkpoint bytes.

AC-05 reserves negotiation capability identifiers for =poll_batch= and
=submit_batch= but does not implement them.  Requesting an unimplemented lane
returns =UNSUPPORTED_CAPABILITY=.  AC-06 owns hot and bulk lanes.

* Handles

A public handle is an opaque fixed-width token logically containing instance
identity, resource identity, generation, and resource kind.  It contains no
pointer and exposes no registry address.

Each resource belongs to one ABI instance and one kind.  Generation checks
prevent ABA when resource identifiers are reused.  Wrong-kind, stale,
cross-instance, and already-released handles are distinguishable typed errors.
Absent an explicitly negotiated concurrency capability, handle operations are
thread-affine and concurrent use is invalid.

* Descriptors and negotiation

Every extensible request and result descriptor begins with =struct_size=.
Semantic fields use fixed-width integers.  Reserved fields must be zero.  No
public layout depends on C enum width, compiler padding, =int=, =long=, pointer
width, or host alignment accidents.

Negotiation exchanges ABI major/minor, Bundle schema, canonicalization profile,
capability bitsets, ownership modes, concurrency profile, batch/payload limits,
runtime identity, and assurance metadata.  Capability identifiers use fixed
width bitsets plus explicit limits, not strings or generic TLV.

Unknown compatible descriptor tails are ignored according to =struct_size=.
Unknown required capabilities, incompatible versions or schemas, non-zero
reserved fields, and malformed sizes fail explicitly.

Bytes and text use pointer plus explicit length.  Text includes an encoding tag.
Digests carry an algorithm tag plus bytes and are not NUL-terminated hexadecimal
strings.  JSON, Scheme datum, implicit NUL scanning, and universal TLV are not
ordinary control-lane representations.

* Ownership

AC-05 supports borrowed inputs and ABI-owned outputs.

- A borrowed input is valid only during the call.  The library cannot retain
  its pointer.
- An ABI-owned output carries its owner handle and is released only through the
  matching v0 operation.
- Failure paths release partial allocations or return a valid releasable owner.
- Double release is detected and reported.

Caller-provided buffers, borrowed leases beyond one call, shared arenas, and
zero-copy claims belong to AC-06.  No Scheme heap address is exposed.

* Error model

Every operation directly returns one typed status.  Diagnostics are explicit
ABI-owned UTF-8 bytes.  There is no global last-error state, ambient =errno=-only
contract, exception crossing, non-local jump, or silent truncation.

Required statuses are:

- =OK=;
- =INVALID_ARGUMENT=;
- =INCOMPATIBLE_ABI=;
- =INCOMPATIBLE_SCHEMA=;
- =UNSUPPORTED_CAPABILITY=;
- =MALFORMED_DESCRIPTOR=;
- =WRONG_HANDLE_KIND=;
- =STALE_HANDLE=;
- =CROSS_INSTANCE_HANDLE=;
- =ALREADY_RELEASED=;
- =INVALID_STATE=;
- =BUNDLE_IDENTITY_MISMATCH=;
- =CHECKPOINT_INCOMPATIBLE=;
- =CANCELLED=;
- =OUTSTANDING_WORK=;
- =ALLOCATION_FAILURE=;
- =INTERNAL_INVARIANT=.

Bundle, epoch, schema, handle, descriptor, and checkpoint mismatches fail
closed.  C engine failures never strengthen Bundle semantics.

* Old graph ABI removal

The same complete feature removes:

- =bindings/runtime-c/include/poo_flow_runtime_abi.h=;
- the old standalone ABI probe;
- old ABI build, install, package, and release-manifest routes;
- Python old-ABI/CFFI bindings and tests that exist only for that ABI;
- public/canonical declarations in old ABI surface, self-test, and benchmark
  documentation.

Python runtime-graph domain code that does not depend on the old ABI remains for
AC-07 migration.  Historical design documents may remain only when marked
retired and cannot be an active contract.

* C harness and qualification

The pure C11 harness covers:

1. successful v0.1 negotiation;
2. version, schema, profile, capability, ownership, and concurrency rejection;
3. Bundle and session lifecycle;
4. idempotent cancel and close disposition;
5. checkpoint round trip, tampering, wrong Bundle, and wrong epoch;
6. wrong-kind, stale, cross-instance, double-release, generation, and ABA;
7. malformed size, non-zero reserved fields, null/length pairs, and overflow;
8. ABI-owned bytes unique release and allocation-failure cleanup;
9. independent public-header compilation without private runtime headers;
10. ASan and UBSan, plus leak sanitizer where supported;
11. deterministic regeneration of header, projection, and vectors;
12. zero remaining old-ABI public/install/build/test routes;
13. AC-01 through AC-04 focused Gerbil regression;
14. qualification receipt field =abi-v1-frozen? #f=.

The installed header and library are sufficient for the harness.  Public types
contain no =___SCMOBJ=, Scheme heap address, Gerbil/Gambit generated symbol,
Python object, C++ class, exception, or mangled name.

* Deferred work

AC-06 owns =poll_batch=, =submit_batch=, caller buffers, borrowed leases,
arenas, native fast views, zero-copy evidence, and the deep
=.data/gerbil-mysql= row/string/buffer/release owner and performance study.  That
study does not block the v0.1 control lane but continues to block ABI v1 freeze.

AC-07 owns the complete production Python CFFI API-mode runtime.  AC-08 and
AC-09 own Cedar and Lean/Evidence integration.  AC-10/11 own full qualification,
canonical cutover, v1 freeze, and removal of pre-v1 surfaces.

* Delivery boundary

Design, implementation plan, schema, generated public artifacts, C engine,
pure C harness, old-ABI removal, build/install updates, owner/status documents,
and qualification receipt form one path-scoped feature commit.  Unrelated
pre-existing build, CLI, CI, or runtime changes remain outside it.
