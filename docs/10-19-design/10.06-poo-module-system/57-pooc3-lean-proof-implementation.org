* 10.06.57.10 Goal

The Lean implementation must prove PooC3 execution semantics, not only manifest
shape.  The first proof milestone is the generic =CanStart= semantics plus a
loop-engine instance with dependency, branch, scope, and policy theorems.

* 10.06.57.30 Lean Theorem Gates

#+begin_src lean
-- Generic PooC3 semantics.
theorem start_sound :
  Step spec state -> CanStart spec state module

theorem deps_completed_of_can_start :
  CanStart spec state module ->
  spec.dependsOn module dependency ->
  spec.completed state dependency

theorem scope_allowed_transitive :
  CanStart spec state module ->
  spec.scopeOrder.le (spec.activeScope state) parentScope ->
  spec.scopeOrder.le (spec.moduleScope module) parentScope

theorem no_sibling_branch_start :
  BranchExclusiveAt spec state ->
  spec.branchSibling left right ->
  CanStart spec state left ->
  CanStart spec state right ->
  False

theorem no_self_dependency_of_rank :
  DependencyRank spec ->
  ¬ spec.dependsOn module module

theorem no_two_cycle_of_rank :
  DependencyRank spec ->
  spec.dependsOn left right ->
  spec.dependsOn right left ->
  False

theorem trace_preserves :
  Preserves spec invariant ->
  Trace spec start final ->
  invariant start ->
  invariant final

-- Loop-engine instance.
theorem choose_strategy_requires_policy :
  CanStart loopSpec state LoopModule.chooseStrategy ->
  loopCompleted state LoopModule.resolvePolicy

theorem runtime_handoff_requires_sandbox :
  CanStart loopSpec state LoopModule.runtimeHandoff ->
  loopCompleted state LoopModule.enforceSandbox

theorem runtime_handoff_requires_full_chain :
  LoopStateConsistent state ->
  CanStart loopSpec state LoopModule.runtimeHandoff ->
  state.profileDone ∧ state.policyDone ∧ state.strategyDone ∧
  state.workflowDone ∧ state.sandboxDone
#+end_src

* 10.06.57.50 Acceptance Criteria

A generated UI case cannot reach runtime handoff unless Lean proves:

#+begin_src latex
\[
  \text{Step}(s,s',m) \Rightarrow \text{CanStart}(s,m)
\]
#+end_src

The =CanStart= evidence must include dependency completion, scope containment,
policy allowance, module precondition, and branch guard.
