#+title: AC-01 Canonical Organization Bundle Implementation Plan
#+startup: overview

* Delivery Policy

AC-01 is implemented in the current project checkout. Intermediate steps are
not committed. One commit is created only after implementation, tests, owner
matrix, receipt, and regression gates are complete. Existing build/CLI-support
changes are never staged or included by AC-01.

* Step 1 — Digest Capability Gate

1. Resolve the installed Gerbil/Gambit SHA-256 API through parser-owned owner
   and dependency evidence.
2. Prove deterministic lowercase hexadecimal output with a focused probe.
3. If no in-process supported API exists, stop and add a bounded internal
   provider dependency; do not use shell commands or process spawning as the
   Bundle identity implementation.

Exit: an in-process =sha256= provider owner and focused test are identified.

* Step 2 — Semantic POO Object Family

Create focused owners under =src/semantic/= for:

- Bundle prototypes and constructors;
- Principal, Role, Agent, capability, delegation, context projection, and tool
  effect semantic objects;
- named slot access and predicates;
- duplicate and ambiguous identity rejection.

Follow the POO object/prototype conventions established by
=src/semantic/boundary-namespace.ss=. Keep public construction POO-native and
keep ordinary data projections internal/read-only.

Exit: the frozen parent/child/tool scenario can be authored only through named
POO helpers.

* Step 3 — Canonical S-Expression v1

Implement pure normalization with:

- fixed schema and field order;
- stable sorting by semantic identity;
- one absence representation;
- symbol/string type preservation;
- rejection of procedures, ports, mutable tables, foreign values, and opaque
  process-local objects;
- idempotent normalization.

Exit: equivalent construction order yields =equal?= canonical data.

* Step 4 — Identity Provider

Serialize canonical S-expression v1 to deterministic bytes, compute SHA-256,
and construct an immutable identity containing algorithm, digest, and epoch.
Unknown algorithms and malformed epochs fail closed.

Exit: equivalent Bundles share identity; every semantic mutation changes it.

* Step 5 — Read-Only Contract Projection

Add a focused =src/contract/= owner that projects canonical payload and Bundle
identity through named accessors. Reuse the pure normalization/accessor style
of the current contract modules without introducing a second authoring DSL or
mutable fact source.

Exit: contract facts agree exactly with the canonical semantic payload.

* Step 6 — Fail-Closed Validator And Receipt

Validate membership, unique identities, parent binding, strict delegation
subset, context visibility, declared typed effect, epoch, digest agreement,
and canonical value stability. Return deterministic ordered diagnostics with
code, semantic path, expected constraint, and observed category.

Exit: ordinary invalid input returns a typed rejection receipt; programmer
contract violations remain explicit POO API errors.

* Step 7 — Frozen Scenario And Negative Matrix

Add focused unit tests and one =t/scenarios/= vertical-slice test covering:

- positive parent/child/restricted-tool Bundle;
- construction-order invariance;
- semantic mutation digest sensitivity;
- normalization idempotence;
- missing and duplicate identities;
- authority equality/escalation;
- context leakage;
- undeclared effect;
- stale epoch;
- digest mismatch;
- unstable semantic values.

Exit: all diagnostic codes and ordering are asserted.

* Step 8 — Regression And Documentation Closure

1. Run the AC-01 focused tests.
2. Reapply/retain the AC-00 tool-calling and runtime-receipt baseline repairs in
   the current checkout and run their focused suites.
3. Run Org lint for RFC 98, this plan, the owner/test matrix, and AC-01 receipt.
4. Update RFC 95 status, owner/test matrix 96, and create the AC-01 receipt.
5. Verify =git diff --check= and ensure only AC-01-owned paths plus the approved
   AC-00 baseline repairs are selected for delivery.

Exit: the complete feature is green and ready for one scoped commit.
