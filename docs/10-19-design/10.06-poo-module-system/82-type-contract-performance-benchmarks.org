* 10.06.82 Type Contract Performance Benchmarks
This document fixes the benchmark boundary for the utilities-backed type contract layer.

The type contract layer is allowed to be rich and strict, but it must not make agent-loop or runtime-handoff hot paths pay repeated full validation and projection costs.

** 10.06.82.10 Benchmark Boundary
Measure Scheme hot-path work:
- =poo-flow-contract-check-slot!= against reusable =PooFlowSlotContract= rows;
- =PooFlowObjectTypeContract -> alist= projection;
- =PooFlowObjectTypeContract -> PooFlowTypeFactContract= projection;
- =PooFlowObjectTypeContract -> PooFlowLeanFactContract= projection;
- real session policy / tool-grant contract projection and slot gates;
- cached projection reads used by doctor, observability, and runtime-handoff presentation.

Do not measure these as type contract overhead:
- cold =gxi= startup;
- package install or ABI refresh;
- C compiler / Apple SDK failures;
- Lean execution;
- Marlin runtime work;
- external IO, subprocess, sandbox, or connector activity.

** 10.06.82.20 Implementation Owners
The benchmark implementation is split by responsibility:
- =t/support/type-contract-performance.ss= owns reusable benchmark workloads.
- =t/type-contract-performance-test.ss= owns regression budgets and receipts.
- =t/performance-tests.ss= includes the suite in the broader performance root.
- =src/testing/project.ss= registers the support file for harness-aware test selection.

The support file is deliberately source-level. It imports the current =src/utilities=, =src/type-facts=, and =src/modules/session= owners directly so benchmark results track the active checkout, not stale installed ABI state.

** 10.06.82.30 Benchmark Classes
Micro benchmark:
- check 10 slots across 2,000 rounds;
- project 10-slot object contracts across 1,500 rounds;
- project type facts and Lean fact contracts across 1,500 rounds.

Scenario benchmark:
- project real session policy type facts;
- project real session tool-grant Lean fact contracts;
- run real session policy and tool-grant =require-slots!= gates.

Scale benchmark:
- check 100 contract-backed objects with 10 slots;
- check 1,000 contract-backed objects with 10 slots;
- check 10,000 contract-backed objects with 2 slots.

Hot-boundary benchmark:
- project 50 slot facts once;
- read the cached fact rows across 10,000 rounds.

** 10.06.82.50 Verification Commands
Use the interpreter-level performance gate for the source hot path:

#+begin_src shell
gxpkg env gxi -e '(begin (import :std/test "./t/type-contract-performance-test.ss") (run-tests! type-contract-performance-test))'
#+end_src

Confirm harness registration:

#+begin_src shell
gxpkg env gxi build.ss spec --tests
#+end_src

Current local =gxc= / package ABI refresh remains a separate toolchain boundary: C compilation can fail on the known Apple SDK/Nix =stdio.h= / =FILE= issue. That failure is not a type contract performance result.

** 10.06.82.60 Design Rule
Type contracts may be strict at declaration, construction, doctor, and handoff boundaries.

Agent-loop hot paths should use cached immutable contract projections unless the object graph changed. Repeated full type-fact or Lean-fact projection inside every loop step is a performance bug unless a specific module proves the graph changed and the projection must be recomputed.
