#+title: AC-08 Kernel Evidence Sink Addendum
#+date: 2026-07-12

* Status

Implemented architecture refinement.  Kernel adapter/evidence vtables, the
two-phase Python synchronous Turso callback, and crash/reopen reconciliation of
finalized and pending reservations are implemented.  ABI v1 remains unfrozen.

* Gap

The kernel-owned adapter vtable now validates and consumes a token before an
effect.  The Python Turso evidence journal rejects replay and root forks, but it
currently appends after native mediation returns.  That is a durable projection,
not the Strict atomic evidence boundary required by RFC 123.

* C ABI contract

- Add a synchronous evidence-sink vtable independent of the effect adapter.
- The evidence sink is two-phase: =reserve= durably records nonce/root/lease
  bindings before the adapter executes; =finalize= records observation and the
  committed or indeterminate root afterward.
- Adapter execution is forbidden unless durable reservation succeeds.  A crash
  after execution but before finalize leaves a recoverable reserved nonce whose
  outcome is Indeterminate and whose execution root does not advance.
- The commit invocation binds semantic root, before root, nonce, session/batch
  sequence, outcome, and observation digest.
- The evidence sink returns the durable after root, evidence reference, kernel
  attestation, and explicit verification flags.
- The kernel advances its execution root only after a successful durable sink
  commit.
- A sink failure after adapter execution consumes the nonce, leaves the root
  unchanged, and returns =Indeterminate=.  It is never a safe retry.
- Borrowed header/payload pointers remain valid only during synchronous calls.

* Claim policy

- L1 means authorization/token mediation is bound.
- L2 requires committed observation and linked execution root.
- L3 additionally requires a durable evidence reference plus independently
  verified kernel signature and inclusion proof.
- Non-empty signature bytes alone never establish L3.
