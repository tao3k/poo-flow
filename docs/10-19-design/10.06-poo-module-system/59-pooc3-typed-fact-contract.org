* 10.06.59 Intent
This document records the implemented typed contract between Scheme fact
emission and Lean proof targets.  It is not a plan document: the concrete
implementation is =PooFlowProof.PooC3.FactContract=.

The design replaces the previous JSON/stringly manifest direction with a
typed Lean boundary.  Scheme still owns external key spelling, but Lean owns
the proof target, target kind, polarity, and well-typedness check.

* 10.06.59.20 Contract Equation
#+begin_src typst
$
"keys"(C_("Lean")) = K_("Scheme")
  "quadandquad"
  forall c in C_("Lean").
    "target"(c) = "expectedTarget"("key"(c))
$
#+end_src

This is implemented by exact key-list theorems plus exact target theorems.
The theorem is intentionally over typed constructors, not JSON objects.

#+begin_src lean
theorem all_contract_keys_exact :
    factKeyContractKeys allFactKeyContracts = allSchemeFactKeys

theorem all_contracts_well_typed :
    ∀ contract,
      contract ∈ allFactKeyContracts ->
      contract.wellTyped

theorem all_contracts_exactly_typed :
    ∀ contract,
      contract ∈ allFactKeyContracts ->
      contract.exactlyTyped
#+end_src

=wellTyped= checks only source, kind, and polarity.  It is deliberately kept
as a weak shape gate.  =exactlyTyped= is the stronger reflection gate: it
requires each Scheme key to map to its specific expected Lean target, so a key
such as =ui.scenario/selector-policy-done= cannot silently point at
=resourcePolicyDone=.

* 10.06.59.30 Typed Surface
The Lean contract is split into focused typed surfaces:

- =SchemeFactKey= names the external Scheme fact key as a constructor.
- =LeanFactTarget= names the Lean proof target as a constructor.
- =FactKeyContract= links key, target, kind, and polarity.
- =UiFailureTheorem= links failure keys to theorem statements and proofs.

#+begin_src lean
inductive LeanFactTarget where
  | sessionFact : SessionLifecycle.SessionFact -> LeanFactTarget
  | scenarioBridgeFact :
      ScenarioProof.ScenarioBridgeFact -> LeanFactTarget
  | uiScenarioFact : UserInterface.UiScenarioFact -> LeanFactTarget
  | uiFailureTheorem : UiFailureTheorem -> LeanFactTarget

def UiFailureTheorem.proof :
    (theoremName : UiFailureTheorem) ->
      theoremName.statement

def UiFailureTheorem.contractKey :
    UiFailureTheorem -> SchemeFactKey

def UiFailureTheorem.missingFact :
    UiFailureTheorem -> UserInterface.UiScenarioFact

def UiFailureTheorem.blockedModule :
    UiFailureTheorem -> UserInterface.UiModule

def SchemeFactKey.expectedTarget :
    SchemeFactKey -> LeanFactTarget
#+end_src

The external string spelling is confined to =SchemeFactKey.externalName=.  It
is an interop label only; it does not decide proof meaning.

External spelling is also checked for duplicate drift:

#+begin_src lean
theorem all_scheme_fact_external_names_nodup :
    allSchemeFactExternalNames.Nodup
#+end_src

* 10.06.59.35 Module Ownership
The contract implementation is intentionally split by proof ownership.  The
public import remains =PooFlowProof.PooC3.FactContract=, but that file is only a
facade over smaller modules.

#+begin_src mermaid
flowchart TD
  Facade["FactContract.lean facade"] --> Aggregate["Aggregate.lean"]
  Aggregate --> Session["Session.lean"]
  Aggregate --> UI["UserInterface.lean"]
  Session --> Target["Target.lean"]
  UI --> Target
  Target --> Failure["Failure.lean"]
  Target --> Base["Base.lean"]
  Failure --> Base
#+end_src

- =Base.lean= owns key/source/external spelling.
- =Failure.lean= owns UI failure theorem statements and dependent proofs.
- =Target.lean= owns Lean targets, expected target, and contract predicates.
- =Session.lean= owns session lifecycle and scenario bridge contract lists.
- =UserInterface.lean= owns UI scenario and UI failure contract lists.
- =Aggregate.lean= owns all-contract composition and global theorems.

* 10.06.59.40 Proof Link
#+begin_src mermaid
flowchart LR
  Scheme["Scheme fact key"] --> Key["SchemeFactKey"]
  Key --> Contract["FactKeyContract"]
  Contract --> Target["LeanFactTarget"]
  Target --> Session["SessionFact"]
  Target --> Bridge["ScenarioBridgeFact"]
  Target --> UIFact["UiScenarioFact"]
  Target --> Failure["UiFailureTheorem"]
  Failure --> Proof["UserInterface projection theorem"]
#+end_src

For =ui.failure/*= keys, the contract is not a name-only mapping.  Each
=UiFailureTheorem= constructor has a dependent =statement= and =proof=, so the
contract compiles only when the corresponding =UserInterface.ui_projection_*=
theorem still exists with the expected type.

* 10.06.59.45 Failure Reflection
UI failure theorem values now expose their reflection data directly:

#+begin_src typst
$
"Failure"(t) =
    angle.l "contractKey"(t), "missingFact"(t), "blockedModule"(t) angle.r
$
#+end_src

The statement is generated from that triple, not repeated by hand for each
failure constructor.

#+begin_src lean
def UiFailureTheorem.statement
    (theoremName : UiFailureTheorem) : Prop :=
  ∀ {facts : UserInterface.UiScenarioFactEnv}
    {activeScope : UserInterface.UiScope},
    (¬ facts theoremName.missingFact) ->
    ¬ CanStart
        UserInterface.uiSpec
        (UserInterface.uiScenarioStateOfFacts activeScope facts)
        theoremName.blockedModule

theorem UiFailureTheorem.blocks_missing_fact
    (theoremName : UiFailureTheorem) :
    theoremName.statement

theorem UiFailureTheorem.required_fact_of_can_start
    (theoremName : UiFailureTheorem)
    {facts : UserInterface.UiScenarioFactEnv}
    {activeScope : UserInterface.UiScope}
    (canStart :
      CanStart
        UserInterface.uiSpec
        (UserInterface.uiScenarioStateOfFacts activeScope facts)
        theoremName.blockedModule) :
    facts theoremName.missingFact

theorem UiFailureTheorem.contract_key_expected_target
    (theoremName : UiFailureTheorem) :
    theoremName.contractKey.expectedTarget =
      LeanFactTarget.uiFailureTheorem theoremName
#+end_src

The next proof layer connects the reflection triple to the actual UI
dependency graph.  A failure cannot pick an arbitrary missing fact: it must be
either a completed module that =blockedModule= depends on, or the benchmark
fixture guard.

#+begin_src typst
$
"missingModule"(t) = "Some"(m)
  => "UiDependsOn"("blockedModule"(t), m)
$

$
"missingModule"(t) = "None"
  =>
    "blockedModule"(t) = "ScenarioBenchmark"
    and "missingFact"(t) = "PerformanceFixtureBound"
$
#+end_src

#+begin_src lean
def uiScenarioFactCompletedModule :
    UserInterface.UiScenarioFact -> Option UserInterface.UiModule

def UiFailureTheorem.requiredBySpec
    (theoremName : UiFailureTheorem) : Prop

theorem UiFailureTheorem.missing_fact_required_by_spec
    (theoremName : UiFailureTheorem) :
    theoremName.requiredBySpec

theorem UiFailureTheorem.benchmark_fixture_guard_reflects_missing_fact
    {facts : UserInterface.UiScenarioFactEnv}
    {activeScope : UserInterface.UiScope}
    (missingFixture :
      ¬ facts UserInterface.UiScenarioFact.performanceFixtureBound) :
    ¬ UserInterface.uiGuard
      (UserInterface.uiScenarioStateOfFacts activeScope facts)
      UserInterface.UiModule.scenarioBenchmark
#+end_src

The blocker proof is now derived from the positive =CanStart= requirement:

#+begin_src typst
$
"CanStart"(s, "blockedModule"(t))
  => "missingFact"(t)
$

$
not "missingFact"(t) =>
  not "CanStart"(s, "blockedModule"(t))
$
#+end_src

This avoids a name-only forwarding table.  The theorem
=required_fact_of_can_start= is the mathematical core; =proof= and
=blocks_missing_fact= are derived from it.
