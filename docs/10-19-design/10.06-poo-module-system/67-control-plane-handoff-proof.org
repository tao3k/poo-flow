#+title: Control Plane Handoff Proof

* Boundary

POO Flow owns policy, profile composition, graph/control-plane contracts, and
formal proof facts. It does not own the runtime executor. Runtime execution is
performed by a downstream runtime such as Marlin or another language runtime.

This boundary is a theorem-level contract:

#+begin_src latex
HandoffAccepted(h) \Rightarrow
  RuntimeOwnerExternal(h) \land ExecutionDeferred(h)
#+end_src

* Handoff Facts

Lean receives a normalized handoff receipt:

#+begin_src lean
structure ControlPlaneHandoffFacts where
  policyReady : Bool
  compositionAccepted : Bool
  graphContractOk : Bool
  runtimeOwnerExternal : Bool
  executionDeferred : Bool
  artifactsDeclared : Bool
#+end_src

The acceptance rule is:

#+begin_src latex
Accepted(h) \equiv
  PolicyReady(h)
  \land CompositionAccepted(h)
  \land GraphContractOk(h)
  \land RuntimeOwnerExternal(h)
  \land ExecutionDeferred(h)
  \land ArtifactsDeclared(h)
#+end_src

* Why This Matters

LangChain and LangGraph compatibility should be demonstrated as control-plane
expressiveness, not by embedding a runtime. A valid POO Flow scenario proves:

- the policy is complete enough to hand off;
- the composition has passed its own proof gate;
- graph facts are accepted;
- runtime ownership is external;
- execution is deferred;
- required artifacts are declared for the runtime owner.

* Rejection Rules

The proof layer rejects a handoff when any required fact is false:

#+begin_src latex
\neg Accepted(h) \Leftarrow
  \neg PolicyReady(h)
  \lor \neg CompositionAccepted(h)
  \lor \neg GraphContractOk(h)
  \lor \neg RuntimeOwnerExternal(h)
  \lor \neg ExecutionDeferred(h)
  \lor \neg ArtifactsDeclared(h)
#+end_src

The two highest-risk rejections are:

- =controlPlaneHandoffRejectedByRuntimeOwner=
- =controlPlaneHandoffRejectedByExecution=

They prevent POO Flow code from drifting into runtime ownership or local
execution semantics.

* Scheme Facts

=src/graph/control-plane-handoff-facts.ss= converts normalized Scheme-side
handoff contract bits into structured proof facts. It does not concatenate Lean
source strings.

#+begin_src scheme
(poo-flow-control-plane-handoff-contract->proof-facts
 'generated-langgraph-control-plane-handoff
 #t #t #t #t #t #t #f)
#+end_src

The Scheme side only emits FFI-ready proof facts. It does not run a graph,
execute tools, coordinate runtime state, or build Lean source text directly.

* FFI Wire Envelope

Structured facts are wrapped by =src/proof/proof-fact-wire.ss= before crossing
the FFI boundary:

#+begin_src scheme
(poo-flow-proof-facts->ffi-wire facts)
#+end_src

The wire envelope is intentionally small:

#+begin_src scheme
((schema . poo-flow.proof.ffi-wire)
 (version . 1)
 (fact-schema . ...)
 (fact-id . ...)
 (accepted? . ...)
 (rejection-rule . ...)
 (fields . ...))
#+end_src

This keeps Scheme responsible for typed control-plane receipts, while the proof
bridge decides how those facts are consumed by Lean or downstream runtime
owners.

The wire layer is schema-aware. Before a fact crosses the bridge, it checks that
the =fact-schema= is known and that all required fields for that schema are
present. For example, =poo-flow.proof.control-plane.handoff= must include:

#+begin_src scheme
(policy-ready
 composition-accepted
 graph-contract-ok
 runtime-owner-external
 execution-deferred
 artifacts-declared
 accepted?
 rejection
 rejection-rule)
#+end_src

This prevents a valid-looking FFI envelope from carrying an incomplete proof
receipt.

* Gate Bundle

The next layer is =src/proof/proof-gate-bundle.ss=. It groups three proof wires:

- composition receipt;
- scenario gap receipt;
- control-plane handoff receipt.

The bundle is accepted only when all three wires are accepted and the runtime
boundary remains true:

#+begin_src latex
GateAccepted(g) \equiv
  CompositionAccepted(g)
  \land ScenarioAccepted(g)
  \land HandoffAccepted(g)
  \land RuntimeBoundaryOk(g)
#+end_src

For LangGraph-like scenarios, the current bundle tests cover:

- accepted composition + accepted scenario + external deferred handoff;
- rejection when runtime ownership is local;
- rejection when the scenario is missing a required capability.
