#+title: AC-08 Batched Merkle Evidence Addendum
#+date: 2026-07-12

* Status

Approved by the AC-08 Batched requirements in RFC 123.  The Scheme/Python
canonical Merkle foundation and shared root/proof vector are implemented.  The
C kernel now negotiates Batched explicitly, returns Buffered without advancing
the root, flushes staged leaves, and restores/flushes them after restart.  The
Python durable journal and synchronous CFFI finalize/flush/reconciliation
projection are implemented.  Batched is never implicit and ABI v1 remains
unfrozen.

* Decision

Batched keeps the write-ahead nonce reservation from RFC 125.  An effect may
execute only after its reservation is durable.  Its observation is then staged
as a durable Merkle leaf and returned as =Buffered=, not =Committed=.  A
separate explicit flush computes one batch root in =O(k)=, persists the root and
proofs, and advances the session execution root once.

* Merkle contract

- A leaf domain-separates schema, leaf index, nonce, first/last runtime
  sequence, payload digest, semantic root, previous execution root, observation
  digest, and outcome.
- The adapter computes =input_digest= while consuming the caller arena and
  returns it beside =observation_digest=.  The kernel projects both into the
  evidence invocation.  This avoids a second kernel payload scan while keeping
  canonical leaf construction inside the evidence sink.
- Hash input uses a language-neutral UTF-8 frame: the domain string followed by
  each canonical field as decimal byte-length, colon, and field bytes.  Public
  fields are canonical strings, symbols, or non-negative integers; language
  printer output and JSON are forbidden hash inputs.
- Internal nodes domain-separate left and right child digests.
- An odd node duplicates the final child at that level.  The rule is canonical
  and shared by Scheme, C vectors, Python, and proof fixtures.
- Tree construction is =O(k)=; inclusion proof length is =O(log k)=.
- The batch receipt binds leaf count, previous root, Merkle root, next execution
  root, crash window, and durable evidence reference.
- The next execution root is canonical
  =SHA-256("poo-flow.batched-execution-root.draft.1|" += framed previous root
  += framed batch root)=.  Both roots use lowercase hexadecimal inside the
  decimal byte-length UTF-8 framing defined above.
- Missing leaves, duplicate indices/nonces, reordered leaves, wrong sibling
  direction, or a different previous root fail closed.

* Runtime contract

- Add =BATCHED_EVIDENCE= as an explicitly negotiated capability.
- Add =Buffered= as a non-final mediation outcome.
- Batched mediation never advances the session execution root.
- Explicit flush requires the expected previous root and the exact staged leaf
  range.  It advances the root only after durable batch commit.
- Strict remains the default.  Diagnostic remains non-executing.
- A runtime without =BATCHED_EVIDENCE= rejects Batched before reservation or
  adapter execution.

* Recovery and claims

- Durable reservations and staged leaves restore consumed nonces after crash.
- Recovery rebuilds the canonical root from staged leaves; it does not execute
  adapters again.
- Buffered receipts are at most L1 mediated.  A verified inclusion proof and
  durable batch receipt can establish L2.  L3 additionally requires verified
  attestation and inclusion proof under the proof policy from RFC 125.

* Qualification

- Scheme deterministic root, odd-leaf, reorder, omission, and proof fixtures.
- Shared Scheme/Python/C vectors for leaf, node, root, and inclusion proof.
- Capability-negotiation and no-implicit-Batched tests.
- Crash/reopen staged-leaf recovery without effect replay.
- Separate Strict and Batched installed-wheel benchmarks.
- =abi-v1-frozen? #f= until Bundle v1 and every AC gate is stable.
