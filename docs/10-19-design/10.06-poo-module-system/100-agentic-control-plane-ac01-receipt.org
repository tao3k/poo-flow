#+title: AC-01 Canonical Organization Bundle Qualification Receipt

* Scope

AC-01 establishes the first executable control-plane artifact: a POO-native,
canonically normalized organization bundle with a versioned S-expression
payload, pluggable digest boundary, SHA-256 identity, read-only contract
projection, and deterministic fail-closed validation receipt.

The feature does not implement a C ABI, Python execution, Cedar evaluation,
Lean proof generation, or downstream Rust/TypeScript runtime adapters.

* Accepted invariants

- The digest is computed by Scheme from the normalized payload.
- Canonical form is =poo-flow.organization-bundle/v1= S-expression data.
- Normalization is deterministic, order-invariant, and idempotent.
- The digest provider boundary is replaceable; AC-01 supplies SHA-256 only.
- Exactly one root agent and one child agent are accepted by this slice.
- Child authority must be a strict subset of the parent authority.
- Context visibility requires an explicit delegation.
- Tool effects must be declared by the selected capability.
- Missing references, duplicates, invalid epochs, identity mismatches, and
  values outside the stable canonical datum domain fail closed.

* Qualification receipt

The focused AC-01 scenario contains eight cases and twenty checks covering:

1. order invariance, idempotence, and SHA-256 identity;
2. the positive frozen one-parent/one-child/restricted-tool bundle;
3. digest change after semantic mutation;
4. rejection of equal parent and child authority;
5. context leakage and supplied-identity mismatch;
6. undeclared tool effect and invalid epoch;
7. duplicate identities and missing references; and
8. fail-closed rejection of unstable canonical values.

The focused AC-01 scenario, tool-calling control-plane scenario, and Gerbil
module compilation pass with the host C include variables removed for each
command.  The LangChain/LangGraph regression is currently blocked by the
pre-existing module-system refactor in the working tree: its composition parser
rejects the new canonical =use-module ... as ...= form, while the composition
smoke test also fails on an unrelated unbound accessor.  AC-01 does not modify
or claim ownership of that refactor.

* Closure

AC-01 is complete at the Scheme semantic and contract boundary.  The next
acceptance slice may project this identity through the performance-oriented C
ABI without changing the canonical payload semantics.
