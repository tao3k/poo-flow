#+TITLE: 10.06.12 Harness Sync Notes

* 10.06.12.01 Sync Goal

The Gerbil harness should learn the stable downstream pattern, not the transient
implementation internals of this repository.

Stable surfaces:

- root-level =user-interface= practice directory;
- =poo-user-module-selection= for hot-plug module choices;
- =poo-settings= for shallow declarative settings;
- =pooUserConfigPresentation= for inspectable downstream receipts;
- package build verification with =gxi build.ss compile=.

* 10.06.12.02 Do Not Sync

Do not publish these as downstream user contracts:

- concrete descriptor realization internals;
- loader backend implementation details;
- merge helper internals;
- runtime adapter execution paths;
- Marlin-owned backend lifecycle code.

* 10.06.12.03 Verification Bundle

Before syncing guidance upstream, record:

- =asp gerbil-scheme check --workspace . --summary=
- =gxi build.ss compile=
- =GERBIL_LOADPATH=.:src:t gxtest t/user-interface-config-test.ss=
- =GERBIL_LOADPATH=.:src:t gxtest t/module-system-user-interface-test.ss=
- full unit test pass for this repository

* 10.06.12.04 Reference Material

Research checkouts under =.data= remain reference corpora.  They should guide
interface design and macro idioms, but they are not runtime dependencies.
