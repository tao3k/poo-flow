* Scope

This audit checks the current Durable implementation against the active RFC and
delivery-plan direction:

- POO-native authoring remains the public Scheme interface.
- Durable runtime work is represented as receipts, manifests, operations, and
  handoff envelopes; Scheme must not execute runtime persistence side effects.
- Runtime-heavy behavior belongs behind Marlin/Python/Turso backend boundaries.
- Session, memory, workflow, artifact, sandbox, and recovery rows must join into
  one durable operation model.

* Evidence Summary

** Implemented And Verified

- Durable policy objects are POO-native and produce bounded runtime manifests.
- Runtime store contracts model fact logs, checkpoints, derived indexes, job
  stores, repair journals, artifacts, communication ledgers, and sandbox ledgers.
- Backend negotiation validates required ledger kinds, capability flags, and
  operation kinds before producing Marlin handoff rows.
- Operation receipts are fixed runtime receipts and project to alists only at the
  handoff boundary.
- Operation bridge covers session graph, communication, memory jobs, workflow
  tasks, artifact refs, and sandbox refs.
- Recovery scenarios validate crash/replay inputs, memory durable jobs, workflow
  rows, sandbox refs, and observability rows.
- Turso backend declaration exists with concurrency, local-first sync, AI vector
  search, libsql vector index, and top-k query capabilities.
- Python runtime dependency metadata declares =pyturso >= 0.6.1=. The
  distribution exposes the top-level =turso= module, so =import turso= is the
  correct runtime import when the project environment is active.
- User-interface custom cases cover durable recovery, runtime store handoff,
  operation bridge, operation receipts, session-memory durable receipts, and
  sandbox durable config.

** Partially Implemented

- The package build gate now passes in develop/debug mode, but the build script
  still reports default =parallelize= from local CPU count. That is package build
  behavior, not a command-line =GERBIL_BUILD_CORES= setting.
