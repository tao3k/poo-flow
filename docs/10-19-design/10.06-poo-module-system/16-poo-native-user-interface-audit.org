* 10.06.16 POO Native User Interface Audit
** 10.06.16.10 Purpose
This note audits the downstream =user-interface/custom/my-module/profiles/*.ss=
shape against the actual POO source used by this repository.

POO-native is a hard project rule, not an optimization target. A user-facing
shape is acceptable only when it lowers into Gerbil POO object composition,
prototype precedence, slot operators, or named POO-native helpers. Raw
alist/record authoring and exposed =(lambda (self super) ...)= compute snippets
are implementation escape hatches, not the recommended interface.

** 10.06.16.20 Local POO Source Evidence
=.data/gerbil-poo/object.ss= and the installed =clan/poo/object= package expose
the object model that =poo-flow= should use directly:
- objects carry supers, slots, defaults, instances, precedence lists, and slot
  functions;
- slot resolution is already backed by C3 through =clan/list=;
- =.def=, =.o=, ==>=, and ==>.+= are the native authoring vocabulary.

=poo-flow= should depend on that instead of reimplementing C3 or presenting a
second profile DSL to users.

** 10.06.16.30 Current Good Landing
=src/modules/sandbox-core/profile-interface.ss= owns public sandbox profile
prototypes and reusable resource prototypes. Backend files such as
=src/modules/nono-sandbox/profile-interface.ss= extend those prototypes with
ordinary Gerbil POO supers.

Downstream files now write this shape:

#+begin_src scheme
(use-module nono-sandbox
  (.def (ci/check @ nono-sandbox-profile
                  network capabilities resources metadata)
    network: (deny-network)
    resources: =>.+ readonly-project-workspace-resources
    metadata: => (lambda (super-metadata)
                   (append super-metadata check-metadata)))

  (.def (ci/build @ ci/check
                  capabilities resources metadata)
    capabilities: => (lambda (super-capabilities)
                       (append super-capabilities
                               '(filesystem-write cache-mount)))
    resources: =>.+ readwrite-project-workspace-resources
    metadata: => (lambda (super-metadata)
                   (append super-metadata build-metadata))))
#+end_src

This is acceptable downstream configuration because the user declares profile
objects, super prototypes, slots, and inherited-slot transforms. It does not run
workflows or realize runtime descriptors.

** 10.06.16.40 Removed Surface
The old sandbox profile row DSL is not public compatibility surface:
- =(use-module nono-sandbox :config (profiles ...))=
- row operators such as =:append=, =:remove=, and =:override=
- raw resource-policy alists as normal authoring
- naked =(lambda (self super) ...)= compute hooks

Those forms can remain only as historical notes, diagnostics, or materialized
ABI/runtime data. The ordinary user and agent path is POO =.def=.

** 10.06.16.50 Current POO Direction
The daily user interface makes POO composition visible at the object level:
- upstream owns profile prototype objects;
- downstream extends named profile prototypes with =.def=;
- direct row operators are replaced by POO slot transforms;
- doctor/presentation output explains resolved prototype chains and field
  sources.

The important boundary is that extension is Gerbil POO object composition, not
another recursive merge DSL.

** 10.06.16.60 Implementation Implications
The current implementation should preserve these constraints:
1. Keep the upstream profile-prototype object layer under
   =src/modules/sandbox-core=.
2. Keep backend sandbox config modules on native =.def= profile objects.
3. Keep doctor output showing prototype chain, direct slots, resolved fields,
   and non-primary authoring diagnostics.
4. Keep row-operator-heavy examples out of the primary downstream path.
5. Keep =:append=, =:remove=, and =:override= only as historical/diagnostic
   vocabulary, not as the recommended user-facing style.

** 10.06.16.70 Decision
Do not rewrite downstream profiles into raw =.o= object expressions or a hidden
wrapper DSL. The public spelling is Gerbil POO =.def= inside =use-module=.

Keep the user interface declarative by exposing upstream profile prototypes and
native slot transforms while the implementation remains native Gerbil POO/C3.
