* 10.06.60.10 Purpose
This document records the LangChain/LangGraph-style user-interface proof case.
The goal is not to clone LangChain or LangGraph. The goal is to show that POO
Flow can express their core policy shapes with a small reusable profile, a lane
policy, and proof-backed lifecycle receipts.

LangChain is used as the reference shape for a linear chain. LangGraph is used
as the reference shape for a guarded state graph. The verification facts remain
owned by POO Flow.
* 10.06.60.20 Core Claim
A small POO profile can express the core loop/graph structure.

#+begin_src latex
\operatorname{LangChainCore}(p)
  \Rightarrow
    \operatorname{LinearChain}(p)
    \land \operatorname{StepDependenciesClosed}(p)
    \land \operatorname{LaneContained}(p)

\operatorname{LangGraphCore}(p)
  \Rightarrow
    \operatorname{GuardedGraph}(p)
    \land \operatorname{FinishReachable}(p)
    \land \operatorname{LaneContained}(p)
#+end_src

This gives users a concrete learning path. They can start from a linear chain,
then move to a guarded graph, then add stronger lane and lifecycle proofs.
* 10.06.60.30 User Interface Shape
The implementation is split into reusable profile fragments and reusable case
fragments:

- =user-interface/profiles/langchain.ss= exposes reusable LangChain-style
  profile slots: =memory=, =prompt=, =model=, =parser=, and =no-tool=.
- =user-interface/cases/langchain-linear.ss= composes those profile slots into
  one production linear chain with fuel =1= and exit =parsed-output=.
- =user-interface/profiles/langgraph.ss= exposes reusable LangGraph-style
  profile slots: =session=, =state=, =router=, =agent-node=, =tool-node=,
  =bounded-loop=, and =runtime-handoff=.
- =user-interface/cases/langgraph-state.ss= composes those profile slots into a
  production guarded state graph with fuel =8=, exit =terminal-edge=, and
  explicit =marlin-control-plane= handoff.

The current implementation is complete for the core reference shapes:

- LangChain-like = linear single-turn chain, no implicit tool branch;
- LangGraph-like = guarded router, explicit loop edges, finish reachability, and
  runtime handoff.
- Production-like runtime receipt = multi-step graph execution trace, loop fuel
  accounting, sandbox/tool containment flags, checkpoint persistence,
  human-approval gate, subagent parentage, and handoff reachability.

It is not a full clone of LangChain or LangGraph. The purpose is to demonstrate
that POO Flow can express the core loop/graph policy shapes with reusable POO
profiles and then verify those shapes through Scheme receipts and Lean facts.
The runtime executor is therefore a POO Flow proof executor: it records graph
semantics and produces receipts; it does not call external tools, models, or
framework schedulers.

* 10.06.60.40 Proof Obligations
The proof obligations are deliberately small and composable.

#+begin_src latex
\operatorname{DependsOn}(\operatorname{ToolStep}, \operatorname{EntryStep})
  \Rightarrow
    \operatorname{Before}(\operatorname{EntryStep}, \operatorname{ToolStep})

\operatorname{DependsOn}(\operatorname{AnswerStep}, \operatorname{ToolStep})
  \Rightarrow
    \operatorname{Before}(\operatorname{ToolStep}, \operatorname{AnswerStep})
#+end_src

For the current user-interface LangChain case this specializes to:

#+begin_src latex
\operatorname{Before}(\operatorname{memory}, \operatorname{prompt})
  \land
\operatorname{Before}(\operatorname{prompt}, \operatorname{model})
  \land
\operatorname{Before}(\operatorname{model}, \operatorname{parser})
  \land
\operatorname{Before}(\operatorname{parser}, \operatorname{parsed\_output})

\neg\operatorname{DependsOn}(\operatorname{parsed\_output}, \operatorname{model})
#+end_src

For graph profiles:

#+begin_src latex
\operatorname{Edge}(a,b)
  \Rightarrow
    \operatorname{Guarded}(a,b)

\operatorname{GraphProfile}(g)
  \Rightarrow
    \operatorname{Reachable}(\operatorname{Finish}(g))
#+end_src

For the current user-interface LangGraph case this specializes to:

#+begin_src latex
\operatorname{Entry} = \operatorname{session}

\operatorname{Finish} = \operatorname{runtime\_handoff}

\operatorname{ConditionalTargets}
  =
\{\operatorname{agent\_node}, \operatorname{tool\_node}, \operatorname{terminal}\}
  \subseteq
\operatorname{Nodes}

\operatorname{LoopEdges}
  =
\{(\operatorname{agent\_node}, \operatorname{router}),
  (\operatorname{tool\_node}, \operatorname{router})\}

\forall n \in \operatorname{Nodes},\,
  \operatorname{CanReach}(n, \operatorname{runtime\_handoff})
#+end_src

For production-like runtime receipts:

#+begin_src latex
\operatorname{ReusableProductionRuntime}(r)
  \Leftrightarrow
    \operatorname{Executed}(r)
    \land \operatorname{Finished}(r)
    \land \operatorname{LoopFuelContained}(r)
    \land \operatorname{HandoffReached}(r)
    \land \operatorname{SandboxScopeContained}(r)
    \land \operatorname{ToolPermissionsContained}(r)
    \land \operatorname{CheckpointPersisted}(r)
    \land \operatorname{HumanApprovalSound}(r)
    \land \operatorname{SubagentsParented}(r)
    \land \operatorname{DiagnosticsEmpty}(r)
#+end_src

For lane policy:

#+begin_src latex
\operatorname{RequestedTool}(l,t)
  \Rightarrow
    \operatorname{AllowedInLane}(l,t)

\operatorname{ReusableLane}(l)
  \Rightarrow
    \operatorname{ScopeContained}(l)
    \land \operatorname{ToolsContained}(l)
#+end_src
* 10.06.60.50 Rejected Cases
The scenario test rejects cases that look plausible but are unsafe:
- a LangGraph conditional branch that targets a missing node;
- a graph that is not finish-total;
- an implicit LangChain tool branch that bypasses the parser/output contract.
- a production runtime trace whose human-approval gate is required but not
  approved.

These examples are important for user education. They show that POO Flow is not
only shorter than a framework clone; it can also reject unsafe shapes before a
policy becomes reusable public surface.
* 10.06.60.60 Implementation Boundary
The Scheme file is a user-interface contribution example. It does not execute
tools, call models, or run a graph scheduler. It emits POO objects and pure
facts.

The scenario owner is =t/scenarios/langchain-langgraph-core-test.ss=. It binds
the reusable profile fragments to the composition macro, creates graph facts,
checks the graph-control receipt metadata, runs the pure graph runtime receipt
executor, and checks the generated Lean fact rows.

The Lean owner is =PooFlowProof.PooC3.LangChainLangGraph=. It proves chain
progression, guarded graph safety, runtime production gates, and lane
containment.

The generated Lean fact owner is
=PooFlowProof.Generated.LangChainLangGraphRuntime=. It is rendered from a
Scheme runtime receipt by
=poo-flow-graph-runtime-receipt->lean-module=. This is the first bridge from
POO Flow runtime receipts to checked Lean source in this case family: Scheme
still owns the receipt, while Lean checks the generated facts against the
production runtime gate.
* 10.06.60.70 Verification Gates
The current verification path has three layers.

Scheme composition and graph receipts:

#+begin_src sh
gxpkg env gxi t/scenarios/langchain-langgraph-core-test.ss
#+end_src

Composition integration:

#+begin_src sh
gxpkg env gxi build.ss test scenario-poo-flow-composition
#+end_src

Lean proof synchronization:

#+begin_src sh
(cd packages/proof/lean && lake build)
#+end_src

Build/cache health should still be checked with the project develop build:

#+begin_src sh
gxpkg build -g
#+end_src

The expected proof evidence is:

- LangChain case: production stage, five composed profile slots, graph payload,
  loop fuel/exit, proof obligations, topological order, empty diagnostics, and
  finish-total receipt.
- LangGraph case: production stage, seven composed profile slots, graph payload,
  bounded loop policy, handoff clause, declared branch targets, explicit loop
  edges, empty diagnostics, and finish-total receipt.
- Broken branch case: undeclared branch target is reported and finish-total is
  false.
- Runtime production case: graph execution trace reaches handoff, consumes loop
  fuel, persists checkpoint, satisfies human approval, contains sandbox/tool
  scope, parents subagents, has empty diagnostics, and auto-generates complete
  Lean fact rows.
- Runtime rejected case: missing human approval leaves the runtime trace
  finished but prevents reusable production status in the generated Lean facts.
- Generated Lean module case: the production runtime receipt renders
  =PooFlowProof.Generated.LangChainLangGraphRuntime=, which defines
  =generatedRuntimeFactRows=, =generatedProductionRuntimeFacts=, and theorem
  witnesses for reusable production runtime facts.
- Lean case: concrete LangChain step order, concrete LangGraph node/edge facts,
  explicit loop-edge theorem, finish reachability, missing target rejection,
  production runtime gate theorems, and generic lane-containment theorems all
  compile.
