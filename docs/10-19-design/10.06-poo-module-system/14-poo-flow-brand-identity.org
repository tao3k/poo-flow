* 10.06.14.01 Decision
=poo-flow= is the brand and module-system identity. Bare =poo= is not retained as a module-system namespace.
This distinction keeps the public surface clear:
- Brand name: =poo-flow=
- Default module group: =poo-flow=
- Scheme owner field: =poo-flow.scheme=
- Module-system owner field: =poo-flow.modules=
- Runtime owner field: =marlin-agent-core=
* 10.06.14.02 Code Rules
Use =poo-flow= in user-visible data:
- presentation =brand-name=;
- default descriptor =group=;
- user config =boundary= metadata;
- doctor and source-doctor metadata;
- documentation and downstream templates.
Canonical public API names use =poo-flow= or =pooFlow=:
- =poo-flow-module=;
- =pooFlowModules=;
- =pooFlowModuleCatalog=;
- =poo-flow-module-active?=;
- =pooFlowEvalModules=;
- =pooFlowModuleSystemPresentation=;
- =poo-flow-user-profile?=;
- =poo-flow-user-config?=.
Old =poo*= spellings are not compatibility aliases for the module system. Public and internal module-system code uses the branded names to avoid split ownership.
