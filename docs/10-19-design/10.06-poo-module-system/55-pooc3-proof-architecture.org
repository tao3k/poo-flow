* 10.06.55.10 Purpose

The proof layer must model PooC3 as control-plane semantics.  C3 decides
local POO precedence.  PooC3 proof then reasons over the graph produced by
composition: scopes, dependencies, branches, policy predicates, strategy
selection, and runtime handoff.

* 10.06.55.20 Core Formula

#+begin_src latex
\[
\begin{gathered}
  \text{C3Order}(M) \neq \text{Safety}(M,\Sigma) \\
  \text{Step}(s,s',m) \Rightarrow \text{CanStart}(s,m) \\
  \text{CanStart}(s,m) \equiv
    \text{DepsCompleted}(s,m)
    \land \text{ScopeAllowed}(s,m)
    \land P_m(s)
    \land Q_m(s)
    \land G_m(s)
\end{gathered}
\]
#+end_src

* 10.06.55.60 Acceptance Rule

A flow is proof-backed only when the emitted strategy step carries Lean
evidence for =CanStart=.  Manifest completeness is only the bridge; it is not
the policy/strategy correctness proof.
