#+title: 10.06.05 Implementation Roadmap

* 10.06.05 Implementation Roadmap
** 10.06.05.05 Current Landing
Status: Marlin facade migration implemented. Slice 1F also landed Nix-style option merge receipts.

The current code does the large-block migration requested from Marlin's Gerbil module surface into =poo-flow/src/modules/module-system.ss=. It includes interface/config construction, structured imports, inline module closure, schema/config projection, validation receipts, module workflow projection, value catalogs, eval receipts, and system presentations.

Not included in this landing: Marlin's concrete policy-pack furniture, deck-runtime dependencies, or Rust handler catalog integration. Those are domain-specific consumers of the module facade, not the =poo-flow= module-system core.
** 10.06.05.10 Slice 1: Descriptor Activation And Inspection
Status: implemented.

Files:
- =src/modules/module-system.ss=
- =t/module-system-test.ss=
- =build.ss=
- =t/unit-tests.ss=

Delivered behavior:
- create and project module source refs;
- create immutable module catalogs;
- resolve selected source refs into module descriptors;
- create POO module descriptors;
- create module descriptors through the thin =poo-module= macro;
- validate explicit imports;
- combine module task-family descriptors into run-config;
- combine module flow-declaration descriptors into run-config;
- expose module activation options with activated module names;
- raise typed =module-system/missing-module-imports= failures;
- raise typed =module-system/missing-module-source= failures;
- report non-executing diagnostics through =poo-module-doctor=.

** 10.06.05.20 Slice 2: Loader Backend
Status: started. First-pass pure loader backend implemented.
Add loader backends that can construct descriptors from source refs before handing them to the existing catalog resolver. This must not change activation semantics.
Planned loader sources:
- local source files;
- package manifests;
- registry manifests;
- generated module descriptors.
Implemented first-pass behavior:
- added =src/modules/loader.ss= as a source-loader owner under =src/modules=;
- added static loader entries for source-ref to descriptor mappings;
- added load receipts so user tooling can inspect loaded and missing sources before activation;
- added =poo-module-load-catalog= to hand loaded descriptors to the existing catalog resolver;
- updated =t/module-system-user-interface-test.ss= to cover loader-to-catalog-to-resolver activation through =:modules/module-system=.
** 10.06.05.30 Slice 3: CLI Or Doctor Command
Status: started. First-pass doctor presentation helpers implemented.
Add a command or helper surface that can explain:
- active modules;
- imports;
- contributed task descriptors;
- contributed flow descriptors;
- missing runtime capabilities;
- config requirements introduced by modules.
This is the =poo-flow= equivalent of borrowing Doom's =doctor= idea without borrowing Doom's implementation.
Implemented first-pass behavior:
- added =src/modules/doctor.ss= as the user-facing doctor presentation owner;
- added =pooModuleDoctorPresentation= for value catalogs;
- added =pooModuleSourceDoctorPresentation= for loader receipts plus module doctor summary;
- included module ids, import graph, contribution counts, diagnostic status, validation count, merge status, and merged option summary;
- kept runtime execution out of doctor projections;
- updated =t/module-system-user-interface-test.ss= to cover both doctor surfaces through =:modules/module-system=.
