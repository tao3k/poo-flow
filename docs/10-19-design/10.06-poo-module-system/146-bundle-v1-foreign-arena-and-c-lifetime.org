#+title: RFC 0146 - Bundle v1 Foreign Arena and C Lifetime
#+startup: overview

* Status

Accepted and implemented.

This RFC defines the Phase 3K boundary that writes an accepted Bundle v1
lowering plan into a checked packed image and transfers that image into one
C-owned, reference-counted native arena.

* Decision

POO Flow owns one runtime interface: the native C ABI. The Scheme control plane
produces a canonical Bundle v1 descriptor and arena image. The C binding copies
that image once into aligned native storage, validates the owned copy, and then
exposes immutable zero-copy views and =O(log n)= component lookup.

Python remains a production-grade runtime used to validate completeness of the
same interface. A Rust kernel, TypeScript runtime, or another downstream
runtime is an adapter over the C ABI; it is not a second runtime implemented by
POO Flow.

The hot path contains no JSON conversion:

#+begin_example
POO objects -> Bundle v1 lowering -> packed native bytes -> C-owned arena
            -> borrowed typed slices -> downstream runtime values
#+end_example

* Ownership boundary

The Scheme writer returns a POO-native foreign-arena image with these owners:

- the accepted Bundle v1 descriptor object;
- a 256-byte packed descriptor u8vector;
- a checked packed arena u8vector;
- the exact arena length;
- structured POO diagnostics when writing fails.

The C creation call does not borrow either Scheme buffer. It validates the
incoming image, allocates 64-byte-aligned C storage, copies the arena exactly
once, copies the descriptor into the handle, and validates the owned result.
The caller may release or move the Scheme buffers immediately after successful
creation.

* Scheme writer contract

The public operation is:

#+begin_src scheme
(feature-bundle-v1-write-foreign-arena lowering-plan)
#+end_src

It accepts one POO-native accepted Bundle v1 lowering plan. It does not accept
a raw alist DSL, JSON document, legacy graph ABI, or an unrelated collection of
rows.

The writer checks:

- descriptor size, schema widths, flags, epoch, and reserved words;
- 16-, 32-, and 64-bit unsigned ranges before each write;
- the 32-byte digest shape;
- region offset, length, stride, row count, and arena bounds;
- component, edge, and evidence row widths;
- 64-byte arena size alignment;
- the current Bundle v1 rule that symbol and metadata payload regions are
  empty.

All integer fields in the packed image use canonical little-endian byte order.
The writer uses functional traversal from =src/utilities/functional.ss= for
byte ranges and row tables. A bounded declarative macro generates the repeated
internal POO object family; the macro expands to ordinary POO-native objects.

Any validation or write failure produces a rejected POO image with a POO
diagnostic. The writer never returns a partially accepted image.

* C ABI

The owning surface is:

#+begin_src c
poo_flow_bundle_v1_arena_create(...);
poo_flow_bundle_v1_arena_create_packed(...);
poo_flow_bundle_v1_arena_retain(...);
poo_flow_bundle_v1_arena_release(...);
poo_flow_bundle_v1_arena_view(...);
poo_flow_bundle_v1_arena_slice(...);
poo_flow_bundle_v1_arena_find_component(...);
#+end_src

=arena_create= accepts a naturally aligned native descriptor.
=arena_create_packed= is the CFFI entry point for byte buffers whose descriptor
alignment is unknown. Packed creation is explicitly little-endian and rejects
an unsupported host byte order instead of interpreting bytes ambiguously.

The opaque handle owns:

- an atomic 64-bit reference count;
- one native descriptor copy;
- the exact arena byte length;
- one 64-byte-aligned arena allocation.

A slice contains a borrowed immutable pointer, byte length, stride, and
alignment. Slice and lookup results remain valid only while the caller holds a
reference protecting the arena handle.

* Lifetime state machine

#+begin_src mermaid
stateDiagram-v2
    [*] --> Incoming: Scheme or CFFI buffers
    Incoming --> Rejected: validation or allocation failure
    Incoming --> Owned: validate, allocate, copy, revalidate
    Owned --> Owned: retain increments atomic count
    Owned --> Borrowed: view, slice, or lookup
    Borrowed --> Owned: borrowed operation completes
    Owned --> Owned: release while count is greater than one
    Owned --> Freed: final release
    Rejected --> [*]
    Freed --> [*]
#+end_src

No borrowed slice extends the handle lifetime automatically. Downstream
adapters must retain before publishing a slice beyond their current call frame
and release that reference when the publication ends.

* Lookup and performance model

Bundle v1 lowering sorts component rows by compact case and component IDs. The
C handle delegates lookup to the existing binary search over the owned
component region. Lookup therefore costs =O(log n)= comparisons and returns a
pointer into the owned arena without allocating or decoding an intermediate
object.

Writing the descriptor is constant work. Writing the arena is =O(n + e + q)=
for =n= component rows, =e= edge rows, and =q= evidence rows. The only ABI
boundary copy is =O(arena bytes)= during handle creation. All subsequent view,
slice, and successful lookup operations are zero-copy.

* CFFI and downstream adapters

The C ABI is the stable language-neutral runtime interface. A CFFI adapter
must:

1. pass the packed descriptor pointer and exact 256-byte length;
2. pass the arena pointer and exact descriptor-declared length;
3. check the returned status and non-null handle;
4. release temporary producer buffers after successful creation if desired;
5. retain the handle for every independently owned runtime reference;
6. expose borrowed slices only under a live handle reference;
7. release the final reference exactly once.

Python may implement all production runtime behavior against this contract.
Rust, TypeScript, and other languages may provide equivalent adapters without
changing Scheme lowering, compact identifiers, table ordering, or lifetime
semantics.

* Fail-closed rules

Creation or writing is rejected when any of these conditions holds:

- the lowering plan is not an accepted POO owner;
- the packed descriptor length is not exactly 256 bytes;
- the packed host byte order is unsupported;
- the schema, flags, digest, reserved words, or region layout is invalid;
- the arena pointer or length disagrees with the descriptor;
- a table is unsorted or contains duplicate keys;
- an allocation fails;
- an atomic retain would overflow.

There is no legacy graph ABI wrapper and no JSON compatibility path.

* Verification

** Phase 3K.1 hardening receipt

The C boundary now verifies the descriptor/arena size before structural
validation, accepts an intentionally unaligned packed descriptor, copies the
descriptor and arena into 64-byte-aligned C-owned storage, and remains stable
after the producer buffers are mutated.  Null retain, null output, incomplete
view, invalid length, and overflow-bound inputs are rejected without exposing
partially initialized ownership.

The cross-language contract is generated for every test run rather than stored
as binary repository state.  A Bazel =sh_test= writes the Scheme descriptor and
arena into =TEST_TMPDIR= through the canonical Scheme foreign-arena writer, then
passes those ephemeral paths to the compiled C harness.  The receipt verifies a
256-byte descriptor, a 576-byte arena, native packed little-endian encoding, and
no hot-path JSON.  Git therefore owns only the producer, consumer, and structural
assertions; it owns no derived ABI image.

The Bazel aggregate owns this Scheme-generation-to-C-consumption test as one
isolated gate rather than leaving either generation or consumption manual.

* Consequences

Phase 3K closes the ownership gap between deterministic Scheme lowering and a
high-performance runtime boundary. Downstream adapters receive typed native
memory with explicit lifetime, stable alignment, checked slices, and logarithmic
component lookup. They do not need to reimplement Bundle semantics or retain
Scheme memory.

The next phase may build production Python CFFI ownership wrappers and broader
runtime adapter qualification directly on this handle without introducing a
second POO Flow runtime or a new serialization format.
