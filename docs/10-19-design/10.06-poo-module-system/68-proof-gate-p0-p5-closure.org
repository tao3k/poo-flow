#+title: Proof Gate P0-P5 Closure

* Scope

This pass closes the first executable proof-gate slice for LangGraph-like
composition scenarios. Runtime execution remains outside POO Flow.

* P0 Boundary

Runtime ownership is represented as proof data, not implementation code:

#+begin_src typst
$
"RuntimeBoundaryOk"(g) equiv
  "RuntimeOwnerExternal"(g) and "ExecutionDeferred"(g)
$
#+end_src

The gate rejects a scenario when runtime ownership is local, even when
composition and scenario capability facts are otherwise accepted.

* P1 Composition Receipt

Composition facts are normalized through:

#+begin_src scheme
(poo-flow-composition-contract->proof-facts ...)
#+end_src

They cross the bridge only through =poo-flow-proof-facts->ffi-wire=.

The next integration layer now accepts structured composition receipts through:

#+begin_src scheme
(poo-flow-composition-receipt->proof-facts receipt)
#+end_src

* P2 Scenario Receipt

Scenario gap facts now use the neutral schema:

#+begin_src scheme
poo-flow.proof.scenario-gap.runtime-row
#+end_src

This supports both accepted and rejected scenario rows.

* P3 FFI Wire

=src/proof/proof-fact-wire.ss= provides a schema-aware wire envelope. It rejects
unknown schemas and facts missing required fields before they cross the bridge.

* P4 Lean Alignment

=PooFlowProof.PooC3.ProofGateBundle= defines the Lean-side acceptance rule:

#+begin_src typst
$
"GateAccepted"(g) equiv
  "CompositionAccepted"(g)
  and "ScenarioAccepted"(g)
  and "HandoffAccepted"(g)
  and "RuntimeBoundaryOk"(g)
$
#+end_src

Generated examples cover accepted LangGraph-like gates, runtime-boundary
rejection, and missing-capability rejection.

* P5 Test Matrix

The current executable matrix covers:

- accepted LangGraph-like proof gate;
- rejection by local runtime ownership;
- rejection by missing scenario capability;
- schema-aware FFI wire validation;
- composition proof facts;
- control-plane handoff facts;
- scenario gap facts.
- receipt-driven proof gate bundles.

The next implementation step is to replace the current LangGraph-like receipt
fixture with receipts emitted by the actual user-interface composition macros.
