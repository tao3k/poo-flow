* Decision boundary

This ledger is the non-destructive entry artifact for AC-11.  AC-10 has closed
the implementation and integrated qualification prerequisites, but RFC 94 P6
also requires an accepted release decision identifying the canonical routes.
That release decision has not yet been recorded.

Therefore this document:

- records the routes that are already qualified to become canonical;
- records retirement candidates and the evidence required before deletion;
- does not freeze ABI v1;
- does not remove shadow code, tests, public exports, or package surfaces;
- does not grant compatibility aliases or a second semantic route.

Explicit migration approval MUST name this ledger (or a successor revision) as
the accepted canonical-route set before any row may move to =remove= or
=publish/freeze=.

* Approval record required for mutation

The P6 release decision must record all of the following in one reviewable
receipt:

1. accepted revision of this ledger;
2. chosen Bundle schema and runtime ABI versions;
3. canonical public route for every retained capability;
4. every row authorized for deletion;
5. rollback as a new validated epoch, never revival of retired semantics;
6. required downstream installed-consumer and absence gates;
7. source revision and AC-10 assurance-manifest identity.

Until that receipt exists, AC-11 state remains =InventoryReadyApprovalBlocked=.

Removal is a dedicated AC-11 feature tranche and MUST NOT be mixed with new
semantic construction.  A failed absence or downstream gate restores the
branch to the pre-cutover revision; it does not add a compatibility wrapper.

* Immediate next gate

Before requesting the release decision, close two parser-owned audits:

- exact public exports and consumers for alist, raw-record, and legacy =.def=
  authoring routes;
- exact Python =ctypes= imports/calls, distinguishing stale documentation and
  diagnostic probes from any production runtime path.

Their results update the =audit= rows in this ledger.  No candidate is removed
as part of that audit.
