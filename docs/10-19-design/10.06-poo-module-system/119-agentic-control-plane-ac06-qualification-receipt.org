#+title: Agentic Control Plane AC-06 Qualification Receipt
#+date: 2026-07-12

* Decision

AC-06 hot/bulk lanes are qualified as a complete pre-v1 feature.  The public
boundary is native C ABI over fixed Scheme-owned types; JSON is not a contract
owner.  =abi-v1-frozen? #f= remains mandatory until Bundle v1 is stable.

* Algorithm receipt

- Pending events form a monotonic ordered window.
- Submit performs one =lower_bound= and a sequential merge: =O(log n + k)=.
- Poll advances a persistent session cursor; acknowledgement compaction adjusts
  that cursor.
- =O(k log n)= per-item lookup is non-conforming.
- All 140 performance blocks report
  =lookup-algorithm=ordered-window-lower-bound= and
  =lookup-complexity=O(log-n-plus-k)=; zero p50 samples: 0.
- Average p50 ns by batch: 1/42, 8/112, 32/315, 128/1326,
  1024/19177.  Crossings per item amortize from 3 to 0.002929688.

* Native ABI and memory receipt

- Generated Scheme-owned contract/header/vector freshness: pass.
- Strict C control, batch, and differential-vector harnesses: pass.
- Fault coverage: malformed struct/reserved fields, invalid stride,
  offset/length overflow, count/stride overflow, bitmap capacity, stale
  generation, early recycle, duplicate submission, and ordered sequence: pass.
- Installed-prefix header plus static-library consumer: pass.
- UBSan control and batch harnesses: pass.
- macOS =leaks= control harness: 0 leaks; batch harness: 0 leaks.
- macOS 26.5 Apple Clang ASan binary still stalls before =main= during runtime
  startup and was interrupted after reproduction.  This is a recorded
  host-toolchain exception; it does not authorize freezing ABI v1.
- Steady-state batch path performs zero per-item allocation and copies no
  payload bytes.

* Runtime and regression receipt

- Real Gerbil dynamic-library FFI control benchmark: pass, 100000 crossings.
- AC-01 through AC-06 exact scenario owners: six suites, all pass.
- Python runtime retained suite: 176 passed.
- The working tree's separate =project-build= CLI refactor no longer accepts
  =build.ss spec --tests=.  Qualification therefore ran the six exact Gerbil
  test owners directly without modifying or including that unrelated refactor.

* Ownership receipt

- Scheme POO objects own the ABI schema and typed runtime projection.
- Generated C declarations and vectors are projections, not sources of truth.
- The C engine owns native handles, caller arenas, leases, ordered windows,
  generation checks, and the hot batch state machine.
- Python remains a production-capable validation runtime; POO Flow exports the
  C binding interface and does not own downstream Rust/TypeScript runtimes.
- Gerbil and gerbil-mysql ownership/release evidence is recorded in RFC 118.
