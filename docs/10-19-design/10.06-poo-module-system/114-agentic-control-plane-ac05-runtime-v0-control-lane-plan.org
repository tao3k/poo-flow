#+title: AC-05 Runtime v0.1 C ABI Control Lane Implementation Plan

* Delivery

Implement the approved v0.1 control lane directly in the current checkout.
Remove the old graph ABI without compatibility aliases.  Preserve unrelated
pre-existing build, CLI, CI, and runtime changes.  Commit only after the complete
AC-05 qualification pack passes.

* Step 1 — Owner and removal ledger

Inventory the old header, probe implementation, build/install routes, Python
CFFI owners, unit tests, ABI self-tests, benchmarks, and active documentation.
Classify Python graph-domain code separately so only old-ABI consumers are
removed.

* Step 2 — Public v0.1 contract

Add the installed C11 header, fixed-width statuses, size-versioned descriptors,
opaque handle tokens, negotiation fields, control-packet envelope, checkpoint
format, ownership descriptors, and capability definitions.  Add a schema owner
and reproducible public artifact/vector generation gate.

* Step 3 — C lifecycle engine

Implement instance, negotiated profile, Bundle, session, checkpoint, restore,
cancel, close, release, owned bytes, diagnostics, generation/ABA protection, and
typed negative paths.  Keep semantic validation in the Bundle projection.

* Step 4 — Pure C qualification

Add an independent C11 harness for negotiation, lifecycle, checkpoint integrity,
descriptor bounds, handle misuse, ownership, allocation cleanup, and public
header isolation.  Run normal, ASan, and UBSan lanes where supported.

* Step 5 — Direct old-ABI cutover

Delete the old graph header and probe, old build/install declarations, old
Python ABI/CFFI bindings and ABI-only tests, and active public contract claims.
Retain Python runtime-graph domain code that does not consume the retired ABI.

* Step 6 — Closure

Run regeneration checks, the C harness, sanitizer lanes, AC-01 through AC-04
Gerbil scenarios, and bounded Python tests for retained surfaces.  Update the
execution plan and owner matrix, write the AC-05 qualification receipt with
=abi-v1-frozen? #f=, inspect a path-scoped diff, and create one complete feature
commit.
