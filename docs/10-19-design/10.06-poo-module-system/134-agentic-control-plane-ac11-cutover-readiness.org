* Purpose

The AC-11 preflight answers one question: is the current revision ready to be
presented for an explicit P6 release decision?  It does not make that decision.
AC-10 receipt 132 remains immutable and is referenced as prerequisite evidence;
its twelve-gate registry is not modified retroactively.

* Contract

The POO-native verifier owner is
=src/qualification/cutover-readiness.ss=.  It requires:

- non-empty AC-10 source revision and manifest digest;
- symbol and version evidence bound to the current source revision;
- accepted JSON runtime-symbol and release-version receipts;
- accepted Python no-=ctypes= and agent-sandbox zero-consumer guards;
- explicit classification of the missing ASP index token/semantic-kind state
  as an external provider blocker rather than a product defect;
- =decision-required? #t=;
- =abi-v1-frozen? #f=;
- =deletion-authorized? #f=.

The accepted receipt means =ready-for-decision= only.  It MUST NOT be consumed
as authorization to publish v1, remove a public route, or delete shadow code.

* Negative gates

The focused qualification rejects:

- symbol or version evidence from another revision;
- rejected/malformed symbol or version receipts;
- missing or failed legacy guards;
- absent or false external-blocker classification;
- bypassed release decision;
- implicit ABI v1 freeze;
- deletion authorization before the P6 decision.

* Status

Focused Scheme tests close the pure verifier contract.  The guarded collector
=tools/run-ac11-cutover-readiness.ss= consumes actual symbol evidence, invokes
the Python no-=ctypes= gate through the Scheme process-memory guard, binds the
AC-10 identity and authoring-audit artifact, and emits one deterministic JSON
=poo-flow.cutover-readiness.v1= artifact.

#+begin_src sh
gxi tools/run-ac11-cutover-readiness.ss \
  SOURCE_REVISION AC10_SOURCE_REVISION AC10_MANIFEST_DIGEST \
  bindings/runtime-c/tests/runtime_v0_exported_symbols.json \
  ACTUAL_SYMBOLS.txt ledger-133-owner-audit
#+end_src

The actual-symbol stream is produced by the same bounded =nm= adapter used by
the runtime C symbol-surface test.  The resulting artifact remains preflight
evidence, not the P6 release decision.
