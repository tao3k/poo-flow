#+title: AC-01 Canonical Organization Bundle Vertical-Slice Design
#+startup: overview

* Status

Approved for implementation on 2026-07-11. This document freezes AC-01 only.
It does not authorize C ABI transport, Python execution, Cedar evaluation, or
Lean theorem expansion.

* Objective

AC-01 establishes one canonical semantic fact for a minimal organization: one
parent Agent delegates exactly one restricted tool capability to one child
Agent. The public surface is POO-native. Pure functions normalize that object
into a versioned canonical S-expression, compute its identity, project a
read-only normalized contract, and validate the frozen scenario fail-closed.

* Decisions

1. The Bundle public API exposes Gerbil POO objects, prototype composition,
   slot access, and named object helpers. Raw alists are not a public authoring
   DSL.
2. The digest input is a versioned canonical S-expression.
3. Bundle identity carries =algorithm=, =digest=, and =epoch=.
4. Digest algorithms are selected through a bounded provider interface. AC-01
   implements only =sha256=.
5. The digest covers semantic canonical payload only. It excludes runtime
   caches, receipts, adapter state, timestamps, and process-local identity.
6. The normalized contract is a read-only projection of the Bundle, never a
   second mutable fact source.

* Frozen Vertical Slice

The accepted Bundle contains:

- one parent Agent and one child Agent;
- one typed Principal and Role for each Agent;
- one explicitly delegable parent tool capability;
- one child authority set that is a strict subset of the parent's delegable
  authority;
- one context projection that excludes data outside child visibility;
- one typed external tool effect;
- one deterministic epoch;
- explicit start, allow, deny, timeout, retry, cancellation, checkpoint,
  restore, stale-epoch, and evidence outcome vocabulary.

AC-01 encodes only the semantic and normalized-contract data required to
represent and validate this slice. Execution state machines remain later work.

** POO Bundle

The Bundle owns semantic membership and relationships. Construction helpers
must reject ambiguous or duplicate semantic identities before normalization.
Prototype extension may add future facets, but extensions cannot replace the
canonical slots defined by AC-01.

** Pure Normalization

Normalization emits ordinary immutable Scheme data. Canonical field order is
fixed by schema version. Unordered collections are sorted by stable semantic
key. Symbols and strings retain distinct types. Procedures, ports, mutable
tables, opaque foreign values, and process-local objects are rejected.

Normalization is idempotent: normalizing a valid normalized projection returns
the same canonical datum.

** Canonical S-Expression v1

The top-level datum identifies its schema and carries ordered sections for
epoch, principals, roles, agents, capabilities, delegations, context
projections, tool effects, and outcome vocabulary. Optional fields have one
canonical absence representation; semantically equivalent omissions and empty
containers cannot produce distinct encodings.

** Digest Provider

The provider interface accepts canonical bytes and returns a lowercase digest
with an explicit algorithm identifier. AC-01 registers only =sha256=. Unknown
algorithms fail closed. Digest computation never accepts an unnormalized
Bundle directly.

** Normalized Contract Projection

The contract projection exposes named accessors and validation facts needed by
later adapters. It contains the canonical payload and Bundle identity but does
not expose mutation, transport framing, or language-specific FFI layouts.

** Validation Receipt

Validation returns a typed receipt with schema, Bundle identity, acceptance,
and ordered diagnostics. Diagnostics carry a stable code, semantic path,
expected constraint, and observed category. Validation does not throw for
ordinary invalid Bundle input; construction/programmer contract violations may
raise at the POO API boundary.

* Validation Rules

The validator rejects:

- missing or duplicate Agent, Principal, Role, capability, or effect identity;
- a child without one parent binding;
- authority not present in the parent's delegable set;
- a child authority set equal to or broader than the parent delegable set;
- context visibility outside the declared child projection;
- an untyped or undeclared external tool effect;
- missing, malformed, or stale epoch identity;
- unsupported digest algorithms or digest/payload mismatch;
- non-canonical or unstable values in semantic slots.

Diagnostics are deterministic and ordered by semantic path then code.

* Test Contract

AC-01 is complete only when:

1. Equivalent Bundles constructed in different orders normalize identically
   and produce the same digest.
2. Every semantic field mutation changes the digest.
3. Normalization is idempotent.
4. The frozen positive parent/child/tool slice is accepted.
5. Missing fields, duplicate identities, authority escalation, context leakage,
   undeclared effects, stale epochs, digest mismatch, and unstable values are
   rejected with stable diagnostic codes.
6. The normalized contract is read-only and agrees with the canonical payload.
7. Existing AC-00 focused Gerbil scenarios remain green.

* File And Ownership Boundaries

The intended owners are:

- =src/semantic/= for Bundle semantics, normalization, identity, and validation;
- =src/contract/= for the read-only normalized projection;
- =t/scenarios/= for the frozen vertical-slice acceptance scenario;
- focused unit tests beside the existing semantic/contract test families.

No AC-01 implementation belongs in =packages/python-runtime/=, C headers,
Cedar adapters, or Lean packages.

* Delivery Rule

Implementation, tests, owner/test matrix updates, and the AC-01 receipt form one
complete feature delivery. Intermediate work is not committed separately.
