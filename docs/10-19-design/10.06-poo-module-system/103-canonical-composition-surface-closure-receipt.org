#+title: Canonical Composition Surface Closure Qualification Receipt

* Result

The composition refactor is closed around one public grammar:

#+begin_src scheme
(use-composition composition-name
  (use-module module-name as alias profile-clause ...)
  (compose composition-profile-ref ...)
  (stage stage-name stage-clause ...))
#+end_src

Legacy module-form tokens, legacy-positive tests, legacy parser branches, and
legacy-specific diagnostics were removed.  Invalid module forms now fail with
the single =composition-invalid-module-form= diagnostic and show only the
canonical grammar.

* Repairs

- The syntax-plan parser uses a fixed four-element prefix plus an ellipsis tail
  for profile clauses, matching Gerbil's list-pattern semantics.
- Grammar literals compare syntax datums rather than lexical bindings, so an
  imported identifier named =as= cannot make a valid form fail hygiene checks.
- =profile-composition.ss= explicitly imports and exports the read-only
  accessors owned by =profile-composition-accessors.ss=.
- Top-level composition profiles are read through
  =poo-flow-composition-profiles=; =compose= is no longer modeled as a stage
  clause.
- LangChain/LangGraph, CrewAI, FunFlow smoke, interface documentation, research
  examples, and the UI README now use the canonical grammar.

Compilation and tests unset the inherited =CPATH=, =SDKROOT=,
=C_INCLUDE_PATH=, and =LIBRARY_PATH= variables per command.  This avoids the
known Nix/Homebrew Gerbil header collision without changing project or global
environment configuration.

profile meaning, stage meaning, or runtime adapter architecture.  Pre-existing
build and CLI-support changes remain outside its commit.
