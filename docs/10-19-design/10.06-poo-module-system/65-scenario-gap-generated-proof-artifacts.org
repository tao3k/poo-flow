#+title: Scenario Gap Generated Proof Artifacts

* Boundary

Scheme does not execute LangGraph or any other long-running runtime. The Scheme
side owns the control-plane receipt contract: plan identity, accepted capability
coverage, rejected capability absence, and the normalized proof facts handed to
Lean.

* Runtime Row Contract

The runtime row is reduced before it enters Lean:

#+begin_src typst
$
"RuntimeRowMatches"(r) equiv
  "planOk"(r) and "rejectionsOk"(r) and "acceptedOk"(r)
$
#+end_src

This is intentionally smaller than the raw receipt payload. Raw payloads belong
to Scheme/Marlin interchange and diagnostics. Lean receives the contract facts
that must be true for a scenario composition to be accepted.

* Positive Generated Artifact

A complete row generates a proof artifact with:

#+begin_src lean
def GeneratedScenarioGapRuntimeRowFacts : ScenarioRuntimeRowFacts where
  planOk := true
  rejectionsOk := true
  acceptedOk := true

theorem GeneratedScenarioGapRuntimeRowMatches :
    runtimeRowMatchesPlan GeneratedScenarioGapRuntimeRowFacts := by
  rfl
#+end_src

This proves that the row satisfies the graph/control-plane contract. It does not
claim the runtime executed; it proves the handoff policy is logically complete.

* Negative Generated Artifacts

Invalid rows must not generate fake positive proofs. They generate rejection
facts instead:

#+begin_src lean
private theorem generatedRuntimeRowRejectedByAccepted
    (facts : ScenarioRuntimeRowFacts)
    (haccepted : facts.acceptedOk = false) :
    not (runtimeRowMatchesPlan facts) := by
  intro h
  unfold runtimeRowMatchesPlan at h
  rw [haccepted] at h
  cases h.right.right

def GeneratedScenarioGapMissingKindRuntimeRowFacts : ScenarioRuntimeRowFacts where
  planOk := true
  rejectionsOk := true
  acceptedOk := false

theorem GeneratedScenarioGapMissingKindRuntimeRowRejected :
    not (runtimeRowMatchesPlan GeneratedScenarioGapMissingKindRuntimeRowFacts) :=
  generatedRuntimeRowRejectedByAccepted
    GeneratedScenarioGapMissingKindRuntimeRowFacts
    rfl
#+end_src

The important property is that every rejected receipt has a proof-facing reason:

#+begin_src typst
$
not "RuntimeRowMatches"(r)
  <=
  not "planOk"(r) or not "rejectionsOk"(r) or not "acceptedOk"(r)
$
#+end_src

This makes missing graph capabilities, invalid plan binding, and rejected
runtime capabilities visible as mathematical facts, not only test failures.

* Current Generated Modules

- =PooFlowProof.Generated.ScenarioGapRuntimeRow= is the positive accepted-row
  artifact.
- =PooFlowProof.Generated.ScenarioGapRuntimeRowRejections= is the negative
  rejected-row artifact covering missing accepted kind, wrong plan, and non-empty
  rejection cases.

* Implementation Hook

=src/graph/scenario-gap-rejection-facts.ss= emits structured proof facts from
normalized runtime contract bits. It does not concatenate Lean source strings.
The facts are FFI-ready data that a proof bridge can translate into the Lean
side:

#+begin_src scheme
(poo-flow-scenario-gap-runtime-contract->proof-facts
 'generated-scenario-gap-missing-kind-runtime-row
 #t
 #t
 #f
 'accepted)
#+end_src

The Scheme side does not inspect runtime internals and does not execute a graph.
It only turns already-derived control-plane facts into typed receipt data. Lean
modules remain Lean artifacts; Scheme should not hand-build them as text.
