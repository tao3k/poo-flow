#+title: AC-06 Batched Hot and Bulk Lanes Implementation Plan

* Delivery

Implement the approved hot/bulk production profile in the current checkout.
Preserve unrelated build, CLI, CI, and runtime changes.  Commit only after the
complete AC-06 qualification pack passes.

* Step 1 — Evidence ledger

Record the exact Gerbil FFI and =gerbil-mysql= pointer, row, string, buffer,
allocation, borrowed-result, and release owners.  Convert the evidence into
explicit arena/lease rules and benchmark controls.

* Step 2 — Generated ABI contract

Extend the runtime v0 JSON contract and generator with layout version,
capability bits, the aligned 96-byte event header, 128-bit compact identities,
arena descriptors, batch requests/results, item statuses, dispositions, and
accepted bitmap rules.  Keep generation freshness mandatory.

* Step 3 — C engine

- Implement the ordered pending window with one =lower_bound= followed by a
  sequential batch merge, for total lookup complexity =O(log n + k)=.
- Keep a persistent poll cursor and preserve it across acknowledgement-prefix
  compaction; reject unordered submit sequences instead of falling back to
  per-item =O(k log n)= lookup.

Implement arena register/release/recycle, lease and generation tracking,
=poll_batch=, =submit_batch=, =batch_ack=, sequence watermarks, duplicate
acceptance, overflow/bounds/overlap checks, frozen input regions, and explicit
capacity reporting.

* Step 4 — Conformance and fault gates

Add pure C and installed-consumer tests for batch sizes, partial acceptance,
stale generation, early recycle, duplicate submission, malformed descriptors,
offset/length/stride/count overflow, bitmap capacity, and zero steady-state
per-item allocation.  Run strict C, UBSan, supported-host ASan, and leak gates.

* Step 5 — Gerbil and differential lane

Add a Gerbil control-plane projection/wrapper that emits typed event/outcome
facts without exposing Scheme heap pointers.  Differentially validate canonical
packets against native headers and reject compact-identity collisions.

* Step 6 — Performance qualification

- Record the lookup algorithm and =O(log n + k)= contract in every benchmark
  block; reject a receipt that reports or implements =O(k log n)= lookup.
- Run the full batch/payload/disposition matrix and prove that all rows use the
  ordered-window algorithm, have non-zero timing samples, and amortize the
  fixed FFI crossings as batch size grows.

Run the required batch, payload, disposition, and path matrix.  Emit
machine-readable receipts with hardware/compiler metadata, crossings,
allocations, copied bytes, GC, latency, throughput, RSS, and hash/seal overhead.

* Step 7 — Closure

Run AC-01 through AC-06 regression, update execution status and owner matrix,
write the qualification and downstream evidence receipts with
=abi-v1-frozen? #f=, inspect the path-scoped diff, and commit one complete
feature.
