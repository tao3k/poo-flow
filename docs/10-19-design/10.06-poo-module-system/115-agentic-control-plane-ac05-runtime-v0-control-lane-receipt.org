* ASan platform evidence

Apple Clang ASan on macOS 26.5 cannot enter the harness.  A sampled process is
stuck before =main= while =libclang_rt.asan_osx_dynamic= initializes shadow
memory through dyld and recursively enters the malloc interceptor spin lock.
This is a host sanitizer-runtime initialization failure, not a sampled C engine
frame.  Homebrew GCC 13 cannot link =libasan= on this host.  The code-level
sanitizer gate therefore uses passing UBSan plus the native =leaks= result.

This exception does not freeze ABI v1.  AC-06/CI must run ASan on a supported
host before any v1 claim.

* Boundary evidence

The public header contains no =___SCMOBJ=, Scheme heap pointer, Gerbil/Gambit
generated symbol, Python object, C++ class, exception, or mangled name.  Handles
contain no pointer.  The C engine validates lifecycle envelopes and never
reimplements Organization, Authority, Context, Protocol, Evidence, Cedar, or
Lean semantics.

The Python runtime no longer publishes =PooFlowRuntimeBinding=,
=PooFlowRuntimeCffiBinding=, =materialize_runtime_graph_plan=, ABI probe scripts,
or the legacy ABI surface manifest.  Graph-domain code remains available without
claiming control-plane ABI conformance.
