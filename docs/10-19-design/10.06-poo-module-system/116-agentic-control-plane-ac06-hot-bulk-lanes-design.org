#+title: AC-06 Batched Hot and Bulk Lanes Design

* Status

Approved in conversation on 2026-07-12.  Implemented and qualified by the AC-06
receipt.  ABI v1 remains deliberately unfrozen; the current macOS ASan startup
limitation is recorded as a host-toolchain exception rather than hidden.

* Objective

AC-06 extends runtime v0.1 with a production batch event lane and bulk payload
lane.  The default production memory model is a caller-owned negotiated native
arena.  Fixed headers carry offsets and lengths into the arena; Scheme heap
pointers, per-field native pointers, implicit strings, and hidden payload copies
are forbidden.

This is a pre-v1 ABI.  Passing AC-06 does not freeze Bundle or ABI v1.

* Owner evidence

The reviewed =.data/gerbil-mysql= owners establish distinct native lifetimes:

- =db/_mysql.scm= defines =MYSQL_RES*= and =MYSQL_FIELD*= as borrowed pointers
  whose lifetime terminates through =mysql_free_result=;
- =MYSQL_BIND*= and primitive result buffers have explicit native allocation
  and release ownership;
- =db/mysql.ss= closes metadata/result scopes with =mysql_free_result=;
- =char-string= paths imply scanning, conversion, and possible allocation.

The reviewed =.data/gerbil/doc/guide/ffi.md= owner distinguishes owned pointers,
borrowed pointers, shallow pointers, arrays, string-member cleanup, and custom
release functions.  It confirms that ownership must be explicit and that a
Scheme foreign object is not itself a stable public buffer contract.

Therefore AC-06 never exposes a Scheme/Gerbil/Gambit object pointer as
zero-copy.  Production zero-copy claims apply only to caller-owned native arena
payload bytes whose address, immutability, generation, and lease are verified.

* Architecture

#+begin_src mermaid
flowchart TD
  Bundle["Session-bound Bundle and identity table"] --> Engine["Runtime v0 C engine"]
  Caller["Caller-owned aligned arena"] --> Register["arena_register"]
  Register --> Lease["Generation lease"]
  Lease --> Poll["poll_batch"]
  Engine --> Poll
  Poll --> Headers["Contiguous fixed headers"]
  Poll --> Slices["Arena offset/length slices"]
  Headers --> Runtime["Downstream runtime"]
  Slices --> Runtime
  Runtime --> Submit["submit_batch"]
  Submit --> Result["Per-item status and accepted bitmap"]
  Result --> Ack["batch_ack"]
  Ack --> Recycle["arena_recycle"]
#+end_src

The C engine owns descriptor validation, sequencing, identity-table lookup,
leases, generation, arena bounds, batch state, duplicate detection, and typed
boundary results.  Bundle remains the semantic owner.  The engine does not
reimplement Authority, Context, Protocol, Evidence, Cedar, or Lean semantics.

* Production memory profile

The default profile is caller-owned negotiated arena:

1. the runtime allocates an aligned contiguous native region;
2. =arena_register= declares address, capacity, alignment, generation, access
   mode, and synchronization profile;
3. the engine returns a generation-bound arena handle and lease;
4. =poll_batch= writes headers and payload slices within declared regions;
5. =submit_batch= reads outcome headers and result slices from the same active
   generation;
6. =batch_ack= terminates accepted item leases;
7. =arena_recycle= advances generation only when no lease or in-flight item
   remains.

Insufficient capacity reports required header count and required payload bytes.
The production path never grows, reallocates, or silently truncates a caller
arena.  ABI-owned batches remain an explicitly negotiated debugging and
low-frequency fallback, not the production profile.

* Hot header profile

The first production layout is 16-byte aligned with a fixed 96-byte stride.  It
contains:

- layout version, event kind, and flags;
- monotonic session sequence;
- 128-bit event identity;
- 128-bit correlation/attempt identity;
- 128-bit authorization-token identity;
- payload offset and payload length;
- monotonic deadline;
- required-evidence bits;
- reserved zero fields.

Bundle digest, epoch, policy digest, and complete identity strings remain bound
to the session and its collision-checked identity table.  They are not repeated
per event.

The 128-bit identities are process-local native fast-view keys derived from full
canonical identities.  Identity-table construction detects collisions and
fails closed.  Full digests remain in canonical packets and evidence.  A header
layout change requires a new negotiated layout version/profile; reserved fields
cannot acquire hidden semantics.

* Payload and authorization integrity

Headers contain offsets and lengths, never per-item payload pointers.  Every
offset, length, stride, count, and capacity calculation is overflow checked and
bounded by the active arena generation.

Payload bytes covered by an authorization decision are frozen from validation
through accepted execution and terminal acknowledgement.  A runtime cannot
mutate checked parameters in place.  Overlapping writable slices, recycling an
in-flight region, stale generations, and mismatched payload digests fail closed.

Text is bytes plus an explicit encoding tag.  The production path contains no
=strlen=, =strdup=, =char-string=, per-field transcoding, or implicit NUL scan.

* Batch API

AC-06 adds:

- =poo_flow_runtime_v0_arena_register=;
- =poo_flow_runtime_v0_arena_release=;
- =poo_flow_runtime_v0_poll_batch=;
- =poo_flow_runtime_v0_submit_batch=;
- =poo_flow_runtime_v0_batch_ack=;
- =poo_flow_runtime_v0_arena_recycle=.

=poll_batch= returns only complete interpretable items.  It reports produced
count, required header count, required payload bytes, lease identity, arena
generation, and sequence range.  No ready work is =OK= with zero produced
items, not an error.

=submit_batch= has two levels of status:

- call-level status covers malformed descriptors, handles, arena generation,
  layout, bounds, and protocol state; a call-level failure consumes zero items;
- per-item status covers identity, sequence, token, payload digest,
  disposition, evidence, duplicate, and runtime outcome checks.

The result includes accepted and rejected counts, an accepted bitmap, per-item
statuses, and the highest contiguous accepted sequence watermark.

* Ordered-window lookup complexity

The hot batch lane MUST preserve monotonic sequence order.  A submit operation
performs exactly one =lower_bound= for the first sequence and then advances by
sequential merge across the ordered pending window.  Its total lookup cost is
=O(log n + k)= for a batch of =k= items over =n= pending events; an
=O(k log n)= per-item binary-search implementation is non-conforming.

Polling maintains a persistent session cursor and walks a contiguous
arena/generation window.  Acknowledgement compaction adjusts that cursor rather
than restarting discovery from the queue head.  The unavoidable =O(k)= term is
the cost of exposing =k= headers, not repeated index lookup.

* Partial acceptance

Batch submission is not all-or-nothing.  External effects cannot be rolled back
as one transaction.

- An accepted attempt is recorded exactly once.
- Resubmitting it returns =DUPLICATE_ACCEPTED= and cannot execute it again.
- An unaccepted item may retry using the same attempt identity.
- Bundle, epoch, token, payload-digest, or identity mismatches reject only the
  item after the batch descriptor itself has passed.
- =indeterminate=, deny, runtime failure, timeout, cancel, and success are
  distinct dispositions.

* Lease and concurrency model

A successful poll creates a lease.  Polled payload bytes remain readable and
authorization-covered bytes remain immutable until acknowledgement.  Result
slices may use a separate writable region in the same active generation.

The default profile is session-thread-affine: the same session cannot be polled
concurrently and poll/submit cannot overlap without a future negotiated profile.
Different sessions may run in parallel when negotiated.  This prevents sequence
races and lock contention in the first production profile.

* Differential validation

Native event/outcome views and canonical packets derive from the same runtime
contract schema.  Qualification proves:

- canonical to native to typed equivalence;
- native to canonical equivalence for every supported kind;
- malformed native views cannot normalize into a different valid event;
- native layout bytes are never persisted or hashed as canonical evidence;
- full identity, Bundle epoch, authorization, and evidence links survive the
  fast-view round trip.

* Performance matrix

Required batches are =1, 8, 32, 128, 1024=.  Required payloads are empty,
=1 KiB=, =64 KiB=, and =1 MiB=.  Required dispositions are allow, deny,
success, runtime failure, timeout, cancel, and indeterminate.

Compared paths are:

- direct C baseline;
- runtime v0 control call;
- single-item batch;
- caller-owned arena batch;
- ABI-owned fallback;
- real Gerbil wrapper crossing;
- string conversion control;
- raw byte-slice path.

Receipts report crossings, allocations, copied bytes, Gerbil GC time, p50, p99,
throughput, RSS, authorization seal/hash overhead, hardware, OS, compiler,
flags, iterations, warmup, and raw machine-readable samples.

* Performance gates

- Per-item boundary crossings decrease as batch size grows.
- The steady-state production arena path performs no per-item malloc/free.
- Production payload copied bytes are zero; writing fixed headers is not counted
  as payload copying.
- No production item path performs string conversion or NUL scanning.
- Arena and batch validation costs remain bounded and are reported separately.
- Any regression requires a reviewed explanation or blocks AC-06.

* Fault and memory qualification

Conformance and property/fuzz fixtures cover malformed offset, length, stride,
count, capacity, bitmap, identity, sequence, status, generation, lease, overlap,
overflow, duplicate submission, early recycle, stale handle, and partial
acceptance.

Required native gates are strict C11 warnings, UBSan, ASan on a supported host,
and platform leak checking.  The macOS 26.5 Apple Clang ASan initialization
failure recorded by AC-05 is not treated as a code pass; another supported host
must supply the ASan receipt before any v1 freeze claim.

* Completion criteria

1. Generated public header and vectors are fresh.
2. Pure C conformance, fault, property, and installed-consumer harnesses pass.
3. Native sanitizer and leak gates pass according to the supported-host matrix.
4. Real Gerbil wrapper benchmarks run; pure C measurements alone are
   insufficient.
5. =.data/gerbil-mysql= ownership/release evidence is recorded.
6. The full performance matrix emits reproducible receipts.
7. AC-01 through AC-06 regressions remain green.
8. The retained production-capable Python runtime suite remains green.
9. Qualification reports =abi-v1-frozen? #f=.

* Deferred work

AC-06 does not implement the Python runtime.  AC-07 uses out-of-line CFFI API
mode to validate the completed control/hot/bulk ABI under production workloads.
Downstream Rust, TypeScript, and other adapters remain downstream-owned.

Cedar token issuance/evaluation and Lean proof generation remain in AC-08/09.
AC-10/11 own complete-chain qualification, canonical cutover, and final v1
freeze.

* Delivery boundary

Design, implementation plan, generated contract changes, C engine, Gerbil
projection/wrapper, conformance/fault harnesses, benchmark matrix, downstream
owner evidence, status documents, and qualification receipt form one complete
path-scoped feature commit.  Unrelated existing build, CLI, CI, or runtime
changes remain outside it.
